Bài 1.5: Cuộc hội thoại đầu tiên

⏱ ~6 phút đọc Level 1 🖥️ Thực hành trên máy
💡
Nguyên lý

Cuộc hội thoại đầu tiên với Claude Code nên đơn giản và thành công. Mục tiêu không phải tạo ra thứ gì ấn tượng — mà là hiểu cách CC hoạt động và xây dựng confidence để tiếp tục. Nhưng trước khi làm bài tập, hãy biết người dùng power dùng CC như thế nào.

Người thực, kết quả thực

Trước khi bạn lo lắng về việc "mình có dùng được không", hãy xem những gì người dùng thực đã đạt được:

🚀
Boris Cherny — Creator của Claude Code

Bản thân người tạo ra Claude Code đã merge 100 Pull Request mỗi tuần khi xây dựng công cụ này. Ông dùng CC để viết code, review, và iterate — liên tục. Đọc workflow của ông tại đây.

✍️
Addy Osmani — Engineering Director tại Google

90% code trong dự án cá nhân của ông được viết bởi Claude Code. Ông dùng CC như co-pilot liên tục, không phải chỉ khi "bí". Chi phí: ~$6/ngày cho một heavy user chuyên nghiệp.

📊
Chi phí thực tế của power user

Heavy user (lập trình viên làm 8h/ngày với CC): ~$6/ngày = ~$180/tháng. User bình thường cho công việc văn phòng: $20–50/tháng. Người mới bắt đầu: $5–15/tháng. Luôn kiểm tra /cost để biết mình đang ở đâu.

Slash commands cần biết ngay từ đầu

Đây không phải danh sách đọc cho vui — đây là những lệnh bạn sẽ dùng trong các bài tập ngay bên dưới:

⚡
/init — Khởi tạo project

Claude đọc cấu trúc folder hiện tại và tự tạo file CLAUDE.md phù hợp. Dùng ngay khi bắt đầu làm việc với project mới — tiết kiệm hàng giờ viết tay.

🗺️
/plan — Plan trước, code sau

Bật Plan Mode: Claude chỉ đề xuất kế hoạch, không tự sửa file. Community consensus: Plan Mode First — luôn xem kế hoạch trước khi cho Claude hành động. Cũng có thể dùng phím tắt Shift+Tab.

📦
/compact — Nén context

Tóm tắt lịch sử hội thoại, giữ nguyên ngữ cảnh quan trọng nhưng giảm 60–80% tokens. Dùng khi conversation đã dài hoặc khi /cost cho thấy bạn đang tiêu nhiều.

💰
/cost — Kiểm tra chi phí

Xem bạn đã dùng bao nhiêu tokens trong session hiện tại. Gõ sau mỗi task lớn để biết task đó tốn bao nhiêu — dần dần bạn sẽ biết cái gì đáng làm, cái gì không.

🔄
/model — Chuyển model

Chuyển giữa Sonnet (mặc định, cân bằng), Opus (mạnh nhất, đắt nhất), Haiku (nhanh nhất, rẻ nhất). Đừng dùng Opus cho tất cả — nó đắt gấp 5x Sonnet.

❓
/help — Danh sách đầy đủ

Xem tất cả lệnh có sẵn và mô tả ngắn. Gõ khi bạn không nhớ lệnh nào đó.

Bài tập thực hành — 3 bước cần làm ngay

Đầu tiên, tạo một folder thử nghiệm an toàn — mọi file CC tạo ra sẽ nằm trong đây, bạn có thể xóa bất cứ lúc nào.

Desktop App: Tạo folder claude-test trên Desktop bằng Finder/File Explorer. Mở Claude Code Desktop → bấm Open folder → chọn folder vừa tạo.

VS Code: Tạo folder claude-test, mở trong VS Code, bấm icon Claude ở sidebar.

Terminal (cho ai đã quen dòng lệnh) 
mkdir ~/claude-test
cd ~/claude-test
claude
1
Bài tập 1: Tạo CLAUDE.md với /init

Mục tiêu: hiểu cách CLAUDE.md hoạt động — file cấu hình được đọc tự động mỗi lần khởi động CC.

Sau khi Claude Code đang chạy trong folder claude-test, gõ:

/init

Claude sẽ tự động tạo file CLAUDE.md dựa trên cấu trúc folder. Mở file đó ra và đọc — đây là "bản hướng dẫn" mà Claude sẽ đọc mỗi khi bạn bắt đầu session mới trong folder này.

Thử thêm 1 dòng vào file: "Tôi đang học Claude Code. Hãy giải thích bằng tiếng Việt và dùng ví dụ đơn giản." — lần sau Claude sẽ tự động biết điều này.

2
Bài tập 2: Plan Mode — Plan trước khi làm

Mục tiêu: học thói quen Plan First — xem Claude dự định làm gì trước khi cho phép nó hành động.

Gõ /plan hoặc nhấn Shift+Tab để vào Plan Mode. Bạn sẽ thấy indicator "[PLAN MODE]" trong giao diện. Sau đó gõ:

Tạo cho tôi một file báo cáo đơn giản tóm tắt 3 kết quả kinh doanh của tháng này.

Quan sát: Claude sẽ chỉ đề xuất kế hoạch — nó sẽ nói "Tôi sẽ tạo file X với nội dung Y" mà KHÔNG tự tạo file. Đây là lúc bạn review và chỉnh sửa nếu kế hoạch chưa đúng ý.

Nhấn Shift+Tab lần nữa để thoát Plan Mode và cho phép Claude thực thi. Thói quen này giúp tránh 80% lỗi "Claude làm sai ý tôi".

3
Bài tập 3: Track chi phí với /cost

Mục tiêu: xây dựng thói quen nhận thức về token usage ngay từ đầu.

Gõ /cost ngay khi bắt đầu session (con số sẽ rất nhỏ hoặc bằng 0). Sau đó làm bài tập 1 và 2 ở trên. Sau mỗi bài tập, gõ lại /cost và ghi nhận:

  • Bài tập /init tốn bao nhiêu tokens?
  • Bài tập Plan Mode tốn bao nhiêu?
  • Tổng cộng session này đến giờ là bao nhiêu cents?

Cuối cùng, gõ /compact để nén context, rồi kiểm tra /cost lần nữa. Bạn sẽ thấy token counter giảm đáng kể — đó là tiền bạn vừa tiết kiệm.

Vòng lặp cơ bản của Claude Code

Sau 3 bài tập trên, bạn đã thấy quy trình Claude Code luôn tuân theo:

  1. CC đọc yêu cầu của bạn và hiểu mục tiêu
  2. CC thông báo kế hoạch — nó định làm gì, dùng Tool nào (Plan Mode)
  3. CC hỏi xác nhận hoặc tự thực thi (tùy chế độ)
  4. CC thực thi và báo cáo kết quả
  5. Bạn kiểm tra và phản hồi nếu cần điều chỉnh

Điểm khác biệt lớn nhất so với Claude Chat: CC không chỉ "nói" — nó làm. File được tạo thật. Lệnh được chạy thật. Kết quả tồn tại sau khi bạn đóng app.

Mẹo từ người dùng thực tế

🎯
Cụ thể về đầu ra

Thay vì "tạo báo cáo", hãy nói "tạo file report.txt chứa bảng tổng hợp doanh thu 12 tháng định dạng markdown". CC cần biết file tên gì, định dạng gì, nội dung gì.

📌
Cung cấp ngữ cảnh

CC không biết bạn là ai, ngành của bạn là gì. 3 câu mô tả ngữ cảnh đầu session thay đổi hoàn toàn chất lượng output. Đây là lý do CLAUDE.md tồn tại.

1️⃣
Một việc mỗi lần

Đừng nhồi 5 yêu cầu vào một câu. Làm tuần tự — xong việc 1 rồi mới hỏi việc 2. CC làm tốt hơn khi có mục tiêu rõ ràng. Và kiểm tra /cost sau mỗi task.

✅
Luôn verify output

"First attempt is 95% garbage" — đây là quan điểm thực tế từ các team engineering chuyên nghiệp. Luôn đọc qua kết quả, mở file được tạo ra, kiểm tra số liệu. Iterate, đừng accept ngay.

🔧
Thực hành thêm

Sau 3 bài tập trên, giao cho Claude Code một việc thực tế từ công việc của bạn: viết một email, tổ chức danh sách công việc tuần này, hay tóm tắt một tài liệu. Trước khi giao, nhớ: (1) bật /plan để xem kế hoạch trước, (2) kiểm tra /cost sau khi xong. Đây là workflow bạn sẽ dùng hàng ngày.

⚠️
Lưu ý an toàn

Claude Code có thể chỉnh sửa và xóa file thật trên máy bạn. Trong quá trình học, hãy luôn làm việc trong folder thử nghiệm riêng (~/claude-test). Không yêu cầu CC thao tác với folder chứa dữ liệu quan trọng khi bạn còn đang làm quen.
