Bài 1.2: Claude Code vs Claude Chat vs API

⏱ ~5 phút đọc Level 1
💡
Nguyên lý

Claude có 3 cách sử dụng khác nhau, phục vụ 3 mục đích khác nhau. Chọn đúng công cụ = hiệu quả gấp đôi. Dùng sai công cụ = tốn thời gian và tiền bạc.

Ba sản phẩm, ba mục đích khác nhau

Nhiều người nghe đến "Claude" là nghĩ ngay đến chatbot trên website. Nhưng Anthropic (công ty tạo ra Claude) cung cấp ba cách tiếp cận hoàn toàn khác nhau. Hiểu sự khác biệt này sẽ giúp bạn không "dùng búa để vặn vít".

Claude Chat
claude.ai — Trò chuyện qua trình duyệt
Bạn mở trình duyệt, vào claude.ai, và gõ câu hỏi. Claude trả lời ngay trong cửa sổ chat. Không cần cài gì, không cần terminal, không cần kỹ thuật.

Tốt cho

  • Hỏi đáp nhanh, brainstorm ý tưởng
  • Viết và chỉnh sửa nội dung (email, bài đăng, báo cáo)
  • Dịch thuật, tóm tắt văn bản bạn paste vào
  • Phân tích ngắn, giải thích khái niệm

Giới hạn

  • Không truy cập file trên máy bạn trực tiếp
  • Không chạy lệnh, không tạo output tự động
  • Không lưu trữ kết quả vào file cho bạn
  • Phù hợp cho câu trả lời — không phải hành động
Claude Code
Trợ lý làm việc trực tiếp trên máy bạn
Claude trở thành trợ lý trực tiếp trên máy tính. Nó đọc file thật, viết file thật, chạy lệnh thật — không chỉ nói chuyện. Dùng qua Desktop App (dễ nhất), VS Code, hoặc Terminal.

Tốt cho

  • Tạo website, landing page, web app hoàn chỉnh
  • Quản lý và tổ chức file hàng loạt
  • Phân tích dữ liệu từ file CSV, Excel
  • Tự động hóa quy trình lặp lại
  • Dự án phức tạp cần nhiều bước và nhiều file

Sức mạnh đặc biệt

  • Đọc và viết file trực tiếp trên máy bạn
  • Chạy lệnh tự động (cài phần mềm, xử lý dữ liệu, deploy...)
  • Tạo output cụ thể (file HTML, CSV, báo cáo)
  • Nhớ ngữ cảnh trong cả session làm việc
Claude API
API — Giao diện lập trình
API (Application Programming Interface) cho phép các developer tích hợp Claude vào ứng dụng của họ. Ví dụ: một công ty muốn xây dựng chatbot chăm sóc khách hàng có sử dụng Claude bên trong — họ dùng API.

Dành cho

  • Developer xây dựng ứng dụng tích hợp AI
  • Tích hợp Claude vào website, app sẵn có
  • Xây dựng workflow tự động phức tạp

Lưu ý

  • Cần biết lập trình để sử dụng
  • Bạn chưa cần quan tâm đến API ở giai đoạn này

Bảng so sánh nhanh

Tiêu chí Claude Chat Claude Code API
Cách truy cập Trình duyệt web Desktop App / VS Code / Terminal Code lập trình
Có thể sửa file? Không Có — file thật trên máy Tùy ứng dụng
Tốt nhất cho Hỏi đáp, viết nội dung Tạo sản phẩm, tự động hóa Xây dựng ứng dụng
Yêu cầu kỹ thuật Không — dùng ngay Không (Desktop App) đến Thấp (Terminal) Cao — biết lập trình

Khi nào nên dùng Claude Code?

Câu hỏi đơn giản để quyết định: Bạn cần KẾT QUẢ CỤ THỂ hay chỉ cần CÂU TRẢ LỜI?

Nếu mục tiêu là hiểu điều gì đó, lấy ý kiến, hoặc viết đoạn văn để bạn copy → dùng Claude Chat, nhanh và tiện hơn.

Nếu mục tiêu là tạo ra file HTML, tổ chức 200 file ảnh, tạo báo cáo tổng hợp từ dữ liệu, xây dựng landing page → Claude Code là lựa chọn đúng vì nó thực sự làm thay bạn, không chỉ nói.

Ví dụ thực tế:

  • "Viết cho tôi email chào hàng cho khách mới" → Claude Chat (bạn copy rồi dùng)
  • "Tạo 50 email chào hàng cá nhân hóa từ danh sách CSV khách hàng và lưu thành file" → Claude Code (tự động, file thật)
  • "Trang web bán hàng tốt nhất trông như thế nào?" → Claude Chat (hỏi để hiểu)
  • "Tạo landing page bán hàng cho sản phẩm X, lưu thành index.html" → Claude Code (tạo file thật)
🔧
Thực hành

Nhìn lại 3 việc bạn đã dùng Claude Chat tuần này. Phân loại: việc nào chỉ cần "câu trả lời" (đúng với Chat), việc nào thực ra bạn cần "file/kết quả cụ thể" (nên dùng Claude Code)? Sự phân biệt này sẽ tiết kiệm nhiều công sức về sau.

⚠️
Lưu ý về chi phí

Claude Chat có gói Free và Pro ($20/tháng). Claude Code tính phí theo token sử dụng — mỗi câu hỏi, mỗi file đọc, mỗi lệnh chạy đều tốn token. Với plan Pro, bạn có credit đủ dùng cho hầu hết công việc thông thường. Chúng ta sẽ nói chi tiết về chi phí ở Bài 1.3.
