Bài 1.4: Khái niệm cốt lõi

⏱ ~7 phút đọc Level 1
💡
Nguyên lý

4 khái niệm nền tảng của Claude Code: Context Window, Tokens, Prompts, và Tools. Hiểu 4 khái niệm này = hiểu 90% cách Claude vận hành. Mọi vấn đề bạn gặp sau này — output kém, chi phí cao, Claude "quên" — đều có thể lý giải qua 4 khái niệm này.

Bạn không cần nhớ thuộc lòng định nghĩa. Bạn chỉ cần hiểu ý nghĩa thực tế của từng khái niệm — nó ảnh hưởng thế nào đến cách bạn làm việc với Claude Code hàng ngày và trực tiếp đến chi phí bạn trả mỗi tháng.

🪙
Tokens
Đơn vị tiền tệ của AI

Token là đơn vị nhỏ nhất mà Claude xử lý và tính tiền. Không phải chữ cái, không phải từ — mà là "mảnh" của từ. Quy tắc thực tế:

  • Tiếng Anh: 1 token ≈ 4 ký tự. "Hello" = 1 token. "Quarterly report" = 3 tokens.
  • Tiếng Việt: tốn 1.5–2x token hơn tiếng Anh do dấu thanh. "Xin chào" ≈ 3–4 tokens. "Báo cáo doanh thu quý 3" ≈ 10–12 tokens.

Pricing thực tế (tháng 3/2026):

  • Claude Haiku 3.5: $0.80 input / $4 output per 1M tokens — nhanh, rẻ, tốt cho tác vụ đơn giản
  • Claude Sonnet 4: $3 / $15 per 1M tokens — cân bằng tốt nhất, dùng cho đa số việc
  • Claude Opus 4: $15 / $75 per 1M tokens — mạnh nhất, chỉ dùng khi thực sự cần

Trung bình 1 session Claude Code xử lý 10K–50K tokens = $0.03–$1.25 mỗi session. Heavy user như lập trình viên chuyên nghiệp tốn ~$6/ngày ($180/tháng). Người dùng bình thường cho công việc văn phòng: $20–50/tháng là phổ biến.

Ví dụ doanh nghiệp: Giống như cước điện thoại quốc tế — tiếng Việt tốn nhiều "phút" hơn tiếng Anh. Nói ngắn gọn, đúng vấn đề tiết kiệm hơn giải thích vòng vo. Nhưng đừng cắt xén ngữ cảnh quan trọng chỉ để tiết kiệm — mất thêm 100 tokens để rõ ràng còn rẻ hơn làm lại cả task.
🪟
Context Window
Bộ nhớ làm việc của Claude

Context Window là tổng không gian Claude có thể giữ trong "đầu" tại một thời điểm. Mọi thứ bạn gõ vào, mọi câu Claude trả lời, mọi file Claude đọc — tất cả đều chiếm không gian này.

Kích thước thực tế:

  • Haiku / Sonnet / Opus: 200,000 tokens (standard)
  • Opus với Max plan: 1,000,000 tokens
  • 200K tokens ≈ 150,000 từ ≈ 500 trang sách — rất rộng rãi cho công việc hàng ngày

Hãy tưởng tượng context window như một bảng trắng có kích thước cố định. Khi bảng đầy, Claude bắt đầu "quên" những thứ cũ nhất — hoặc output trở nên kém chất lượng khi context quá tải.

Ý nghĩa thực tế: bắt đầu conversation mới khi chuyển sang task hoàn toàn khác. Đừng dùng một session cho cả ngày làm việc với nhiều chủ đề khác nhau.

Ví dụ doanh nghiệp: Giống như RAM máy tính — 16GB RAM đủ để chạy nhiều phần mềm, nhưng nếu bạn mở 50 tab Chrome, 5 video YouTube, và 3 phần mềm nặng cùng lúc, máy vẫn chậm. Context window cũng vậy — rộng nhưng không vô hạn.
💰
Quản lý Token — Tiết kiệm thực tế
4 chiến lược người dùng chuyên nghiệp dùng

Dùng Claude Code nhiều mà không quản lý token = lãng phí tiền không cần thiết. Đây là những gì người dùng thực sự tiết kiệm 40–70% chi phí:

  • /compact — nén conversation history: Giảm 60–80% tokens trong context bằng cách tóm tắt lịch sử hội thoại. Dùng sau 10–15 tin nhắn hoặc khi bạn thấy context đã dài.
  • Bắt đầu conversation mới cho task mới: Đừng gộp "viết email + phân tích data + làm website" vào 1 session. Mỗi task = 1 session riêng. Mang lại output tốt hơn và tốn ít token hơn.
  • CLAUDE.md ngắn gọn (dưới 300 dòng): File CLAUDE.md được đọc tự động MỖI LẦN bắt đầu session mới. File 500 dòng ≈ 5,000 tokens lãng phí mỗi session, mỗi ngày. Mỗi dòng trong CLAUDE.md phải thực sự cần thiết.
  • Prompt caching tự động: CLAUDE.md được cache sau lần đọc đầu tiên — các session sau tiết kiệm 90% chi phí đọc lại file này. Cache tự động, bạn không cần làm gì thêm.

Model routing thực tế: Dùng Sonnet cho hầu hết việc. Chuyển sang Opus chỉ khi task cần reasoning phức tạp (phân tích chiến lược, debug khó). Dùng Haiku cho task đơn giản (format data, trả lời ngắn). Opus đắt gấp 5x Sonnet — đừng dùng Opus để viết email.

Lệnh /cost: Gõ lệnh này bất cứ lúc nào để xem bạn đã tiêu bao nhiêu tokens trong session hiện tại. Dùng như đồng hồ đo điện — giúp bạn biết task nào tốn nhiều nhất và tối ưu theo đó.
📝
Prompts
Cách bạn giao việc

Prompt là những gì bạn gõ để giao việc cho Claude. Chất lượng prompt quyết định chất lượng kết quả — có lẽ là yếu tố quan trọng nhất trong toàn bộ trải nghiệm dùng Claude Code.

Prompt tốt không cần phức tạp. Nó cần cụ thể, có ngữ cảnh đủ, và rõ ràng về đầu ra mong muốn.

  • Mơ hồ: "Làm cho tôi một website đẹp"
  • Tốt hơn: "Tạo landing page giới thiệu khóa học marketing online cho SME, có hero section với headline, 3 lợi ích chính, và form đăng ký email. Dùng màu xanh navy và cam. Lưu thành file index.html."

Kỹ năng viết prompt là kỹ năng quan trọng nhất bạn sẽ học. Chúng ta sẽ thực hành nhiều ở các Level sau.

Ví dụ doanh nghiệp: Brief mơ hồ → nhân viên làm sai → mất thời gian cả hai. Brief rõ ràng → nhân viên làm đúng ngay lần đầu. Với Claude, chi phí làm lại còn bao gồm cả tokens — nên đầu tư vào prompt tốt từ đầu là tiết kiệm nhất.
🛠️
Tools & Slash Commands
Khả năng hành động và điều khiển

Tools là các "tay chân" của Claude Code — khả năng thực sự làm điều gì đó ngoài nói chuyện. Claude Code có đầy đủ bộ công cụ để làm việc với file, web, và hệ thống:

  • Read / Write / Edit — đọc, tạo, chỉnh sửa file trên máy bạn
  • Bash — chạy bất kỳ lệnh terminal nào (npm, git, python, ffmpeg...)
  • Glob / Grep — tìm file và tìm kiếm trong code theo pattern
  • WebSearch / WebFetch — tìm kiếm web và đọc nội dung trang (khi được bật)
  • MCP tools — kết nối với dịch vụ bên ngoài (database, API, Slack...) qua Model Context Protocol

Slash commands quan trọng cần biết ngay:

  • /init — tạo CLAUDE.md tự động từ cấu trúc project hiện tại
  • /plan — chế độ planning, Claude chỉ đề xuất không tự sửa code
  • /compact — nén context, giảm token usage 60–80%
  • /cost — xem chi phí token của session hiện tại
  • /model — chuyển đổi giữa Sonnet / Opus / Haiku
  • /help — danh sách đầy đủ lệnh có sẵn
Shift+Tab: Phím tắt để bật/tắt Plan Mode ngay trong terminal — Claude chỉ plan, không thực thi. Dùng khi bạn muốn xem Claude dự định làm gì trước khi cho phép nó hành động.

Cách 4 khái niệm kết nối với nhau

Đây là luồng xảy ra mỗi khi bạn gõ một lệnh cho Claude Code:

Bạn gõ Prompt
→
CC tiêu tốn Tokens để xử lý
→
CC chọn Tool phù hợp
→
Kết quả lưu vào Context Window
→
Bạn xem và phản hồi

Hiểu luồng này giúp bạn biết tại sao Claude đôi khi "không nhớ" điều gì đó (context window quá tải), tại sao một số yêu cầu tốn chi phí cao hơn (nhiều token), tại sao prompt rõ ràng quan trọng (định hướng CC chọn Tool đúng và tránh làm lại).

🔧
Thực hành ngay

Mở Claude Code, gõ /cost để xem token usage hiện tại. Hỏi một câu bất kỳ, rồi gõ /cost lại và quan sát con số thay đổi — đó là tokens vừa được tiêu tốn. Tiếp theo, gõ /compact để nén context, rồi kiểm tra /cost lần nữa. Quan sát sự khác biệt trước và sau compact.

⚠️
Cảnh báo CLAUDE.md

CLAUDE.md dài 500 dòng = lãng phí ~5,000 tokens MỖI SESSION. Với 10 session/ngày = 50,000 tokens/ngày tiêu tốn chỉ để đọc file config. Nhân 30 ngày = 1.5 triệu tokens/tháng chỉ từ CLAUDE.md dài. Giữ dưới 300 dòng. Mỗi dòng phải earn its place.
