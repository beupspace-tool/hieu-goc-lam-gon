Bài 1.8: Bài tập thực hành

⏱ ~7 phút đọc Level 1 🖥️ Cần thực hành trên máy
💡
Nguyên lý

Đọc lý thuyết giúp bạn hiểu. Thực hành giúp bạn nhớ. 3 bài tập dưới đây được thiết kế để củng cố những gì bạn học trong Level 1 — đồng thời tạo ra kết quả thực tế có thể dùng được trong công việc.

Mỗi bài tập có mục tiêu rõ ràng, bước thực hiện chi tiết, và tiêu chí kiểm tra kết quả. Hãy làm đúng thứ tự — bài sau xây dựng trên bài trước.

Bài tập 1
Tổ chức folder dự án
⏱ ~15 phút 🎯 Kỹ năng: Quản lý file, Prompt cụ thể
🎯 Mục tiêu

Tạo cấu trúc folder chuẩn cho một dự án website — đồng thời thực hành cách mô tả yêu cầu cụ thể cho Claude Code.

📋 Bước thực hiện
  1. Mở Terminal, tạo folder mới và vào trong đó: 
    mkdir ~/du-an-website && cd ~/du-an-website && claude
  2. Khi Claude Code khởi động, gõ prompt sau (thay tên công ty bằng tên thật của bạn): 
    Tạo cấu trúc folder hoàn chỉnh cho dự án website công ty [TÊN CÔNG TY].
Cần có các thư mục:
- docs/ (chứa tài liệu dự án)
- images/ (ảnh sản phẩm và banner)
- reports/ (báo cáo hàng tháng)
- templates/ (template email và tài liệu)
Sau đó tạo file README.md trong thư mục gốc với nội dung giới thiệu ngắn về dự án và mô tả mục đích của từng thư mục.
  3. Sau khi CC tạo xong, mở Finder/File Explorer và kiểm tra cấu trúc thực tế.
  4. Mở file README.md và đọc nội dung — điều chỉnh nếu chưa phù hợp bằng cách yêu cầu CC sửa.
✅ Kết quả mong đợi
Bạn có folder du-an-website với 4 thư mục con và 1 file README.md. Mở README.md — nội dung phải mô tả đúng mục đích từng thư mục. Nếu nội dung quá generic, thử thêm context: "Đây là website bán [sản phẩm của bạn] cho [đối tượng khách hàng]."
💬 Gợi ý nếu gặp khó khăn
CC không tạo thư mục? Hãy kiểm tra bạn đang ở đúng thư mục bằng lệnh pwd. Sau đó thử lại với prompt đơn giản hơn: "Tạo thư mục docs, images, reports, templates trong thư mục hiện tại."

README.md nội dung không đúng? Yêu cầu CC sửa với nhiều context hơn: "Sửa lại README.md. Tôi là [nghề nghiệp], công ty [tên], dự án này để [mục đích]."
Bài tập 2
Viết email chào hàng
⏱ ~20 phút 🎯 Kỹ năng: Tạo nội dung, So sánh output
🎯 Mục tiêu

Tạo 3 phiên bản email chào hàng với tone khác nhau, lưu thành file riêng biệt. Thực hành cung cấp context cụ thể và so sánh chất lượng output.

📋 Bước thực hiện
  1. Tiếp tục trong phiên Claude Code đang mở, hoặc mở mới trong folder du-an-website/templates.
  2. Gõ prompt sau — điền đầy đủ thông tin thật của bạn: 
    Tôi là [tên bạn], [chức danh] tại [tên công ty].
Chúng tôi cung cấp [mô tả sản phẩm/dịch vụ ngắn gọn].
Tôi muốn gửi email chào hàng đến [mô tả khách hàng mục tiêu].

Viết 3 phiên bản email chào hàng khác nhau:
1. email-trang-trong.txt — phong cách trang trọng, chuyên nghiệp
2. email-than-thien.txt — phong cách thân thiện, gần gũi
3. email-ngan-gon.txt — rất ngắn, dưới 100 chữ, thẳng vào vấn đề

Lưu mỗi phiên bản thành file riêng trong thư mục templates/.
  3. Sau khi CC tạo xong, mở từng file và đọc kỹ.
  4. Chọn phiên bản bạn thích nhất. Yêu cầu CC tinh chỉnh: "Tinh chỉnh email-than-thien.txt: thêm câu hỏi cuối email để khuyến khích phản hồi."
✅ Kết quả mong đợi
Bạn có 3 file email trong thư mục templates. Mỗi file có tone rõ ràng khác nhau. File "ngắn gọn" phải dưới 100 chữ thực sự. Nội dung phải đề cập đúng sản phẩm/dịch vụ của bạn — không phải nội dung generic.
💬 Gợi ý nếu gặp khó khăn
Email quá generic không liên quan đến business? Thêm nhiều context hơn: đối thủ của bạn là ai, điểm khác biệt của bạn là gì, pain point của khách hàng là gì.

3 phiên bản trông giống nhau? Yêu cầu cụ thể hơn về sự khác biệt: "Phiên bản thân thiện: dùng từ ngữ đời thường, kể một câu chuyện ngắn, không có bullet points."
Bài tập 3
Tóm tắt & phân tích tài liệu
⏱ ~20 phút 🎯 Kỹ năng: Phân tích dữ liệu, Đọc file
🎯 Mục tiêu

Thực hành khả năng đọc và phân tích tài liệu của Claude Code. Tạo báo cáo tóm tắt từ dữ liệu có sẵn.

📋 Bước thực hiện
  1. Đầu tiên, tạo tài liệu mẫu để CC phân tích. Bạn có thể dùng nội dung thật từ doanh nghiệp (báo cáo doanh thu, mô tả sản phẩm, feedback khách hàng) hoặc dùng ví dụ sau. Yêu cầu CC tạo file mẫu: 
    Tạo file bao-cao-quy3.txt trong thư mục reports/ với nội dung báo cáo kinh doanh quý 3 giả định cho một công ty dịch vụ thiết kế. Bao gồm: doanh thu 3 tháng (7, 8, 9), số dự án hoàn thành, top 3 loại dịch vụ, và 2-3 thách thức gặp phải trong quý.
  2. Sau khi file được tạo, yêu cầu CC phân tích: 
    Đọc file reports/bao-cao-quy3.txt và tạo file reports/tom-tat-quy3.txt với nội dung:
1. Tóm tắt tình hình kinh doanh trong 3 gạch đầu dòng
2. Điểm mạnh nhất trong quý (với số liệu cụ thể)
3. Vấn đề cần chú ý nhất
4. 2 đề xuất hành động cho quý 4
  3. Mở file tom-tat-quy3.txt và đọc bản tóm tắt.
  4. Thử thêm một bước nâng cao: "Dựa trên file báo cáo này, viết thêm vào cuối file tom-tat-quy3.txt một đoạn ngắn dự báo quý 4 nếu xu hướng hiện tại tiếp tục."
✅ Kết quả mong đợi
Bạn có 2 file trong thư mục reports: file báo cáo gốc và file tóm tắt. File tóm tắt phải rõ ràng hơn, dễ đọc hơn, và có 4 phần đúng như yêu cầu. Nếu CC phân tích đúng dữ liệu trong báo cáo — điểm mạnh nổi bật là thành công.
💬 Gợi ý nếu gặp khó khăn
Muốn thử với dữ liệu thật? Copy nội dung từ báo cáo thật của bạn vào một file .txt trong thư mục reports, sau đó yêu cầu CC tóm tắt file đó. CC sẽ phân tích dữ liệu thật của bạn.

Bản tóm tắt quá ngắn hoặc quá dài? Chỉ định rõ: "Mỗi phần không quá 3 câu" hoặc "Gạch đầu dòng, không viết paragraph."

Bonus Challenge

Nếu bạn hoàn thành cả 3 bài tập, hãy thử thêm: yêu cầu CC tạo một file ket-qua-level-1.md tóm tắt những gì bạn đã làm được trong 3 bài tập — bao gồm danh sách file đã tạo, kỹ năng đã thực hành, và một điều bạn muốn học thêm. Đây là cách dùng CC để ghi lại hành trình học của chính bạn.

Bạn đã có nền tảng vững chắc!
