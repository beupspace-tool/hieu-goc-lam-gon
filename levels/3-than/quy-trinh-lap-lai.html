Bài 3.5: Từ task lẻ sang quy trình

⏱ ~8 phút đọc Level 3 Workflow
💡
Nguyên lý

Dùng Claude Code cho một task một lần là tốt. Nhưng giá trị thực sự đến khi bạn xây workflow tái sử dụng — cùng quy trình, mỗi tuần hoặc mỗi tháng, chạy nhất quán mà không cần bạn nhớ từng bước. Đây là sự khác biệt giữa "người dùng Claude" và "người vận hành hệ thống".

Tại sao cần thiết kế quy trình

Hầu hết người dùng Claude Code ở giai đoạn đầu làm như sau: nhớ ra cần làm gì → mở Claude Code → gõ prompt ngẫu hứng → nhận kết quả → repeat tuần sau với prompt khác một chút. Không nhất quán. Phụ thuộc vào "mood" của bạn hôm đó.

Một workflow được thiết kế tốt trông khác hẳn: bạn có file prompt templates sẵn, bạn biết chính xác phải chạy lệnh gì theo thứ tự nào, output được lưu đúng chỗ, và kết quả tuần này có thể so sánh được với tuần trước. Không phụ thuộc vào trí nhớ hay cảm hứng.

4 bước xây quy trình

1

Capture — Ghi lại workflow thủ công

Trước khi tự động hóa bất cứ điều gì, ghi lại chính xác bạn đang làm gì thủ công. Inputs là gì? Outputs là gì? Bao nhiêu bước? Bước nào mất thời gian nhất? Bước nào hay bị quên? Đây là "as-is process" — không cần đẹp, cần thật.

2

Template — Tạo prompt templates

Mỗi bước trong workflow → một prompt template. Templates không phải câu hỏi mở — chúng là instructions cụ thể với placeholders cho data thay đổi mỗi lần. Lưu vào CLAUDE.md hoặc folder templates/ trong project. Tốt nhất: tạo file workflow-weekly-content.md chứa tất cả prompts theo thứ tự.

3

Automate — Dùng hooks và scripts

Hooks là event listeners: Claude Code phát sự kiện khi xảy ra action → hook chạy script tương ứng. Ví dụ: mỗi khi Claude ghi file vào drafts/, hook tự động chạy spell-checker. Mỗi khi session bắt đầu, hook tạo backup. Automations nhỏ này cộng lại tiết kiệm nhiều thời gian.

4

Iterate — Cải thiện qua từng lần chạy

Sau mỗi lần chạy workflow, dành 5 phút ghi chú: bước nào còn awkward? Output nào chưa đúng format? Prompt nào cần rõ hơn? Update templates ngay. Sau 4-6 lần, workflow sẽ chạy gần như tự động và output sẽ đạt chất lượng nhất quán.

Hooks — event automation

Hooks được cấu hình trong CLAUDE.md hoặc file .claude/hooks.json. Claude Code hỗ trợ các events:

  • PreToolUse: Chạy trước khi Claude dùng tool (đọc file, ghi file, chạy lệnh)
  • PostToolUse: Chạy sau khi Claude dùng tool
  • SessionStart: Chạy khi bắt đầu session mới
  • SessionEnd: Chạy khi kết thúc session

Ví dụ hook cấu hình trong .claude/settings.json:

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'File đã được lưu: $TOOL_INPUT_PATH' >> .claude/activity.log"
          }
        ]
      }
    ],
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "git add . && git commit -m 'auto: pre-session backup' 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}

Hook đầu tiên log mọi file Claude ghi. Hook thứ hai tự động commit backup mỗi khi bạn bắt đầu session mới với Claude Code.

Ví dụ thực tế: Weekly Content Workflow

Thứ 2

Claude đọc content calendar từ content-calendar.csv → draft 3 blog post outlines dựa trên chủ đề tuần này → lưu vào drafts/week-XX/outlines.md

Thứ 3

Bạn review outlines, để lại comments trong file → Claude viết full drafts dựa trên approved outlines → lưu 3 file riêng trong drafts/week-XX/

Thứ 4

Claude tạo social posts từ mỗi blog: 1 post cho Facebook (800-1200 ký tự), 1 cho LinkedIn (tập trung insight), 1 cho Instagram (caption + hashtags), 1 cho X (280 ký tự), 1 cho Threads → lưu vào social/week-XX.md

Thứ 5

Claude tạo email newsletter từ 3 blog posts: executive summary, top 3 insights, CTA → lưu vào email/week-XX-newsletter.html

Thứ 6

Final review toàn bộ content → bạn approve → schedule trên Buffer hoặc gửi email newsletter

Mỗi ngày, thay vì nghĩ "hôm nay làm gì", bạn mở Claude Code, chạy prompt template đúng ngày, và work với output Claude tạo ra. Tổng thời gian bạn cần: 30-45 phút/tuần thay vì 4-6 giờ.

Ví dụ thực tế: Monthly Report Workflow

# Prompt template: monthly-report.md
# Chạy vào đầu mỗi tháng

Bối cảnh: Tôi là {tên bạn}, {vị trí} tại {công ty}.
Dữ liệu input nằm trong folder data/reports/{tháng}/

Task:
1. Đọc tất cả CSV files trong data/reports/{tháng}/
2. Phân tích: doanh thu, số đơn hàng, top 5 sản phẩm, so sánh với tháng trước
3. Identify 3 điểm đáng chú ý (tích cực và tiêu cực)
4. Tạo executive summary 5 bullet points
5. Lưu báo cáo vào reports/{tháng}-report.md với format chuẩn (xem template trong templates/monthly-report-template.md)

Không thêm nhận xét ngoài phạm vi data. Nếu data thiếu, ghi rõ "Không đủ data" thay vì estimate.

Prompt này không thay đổi tháng nào. Bạn chỉ cần đảm bảo data CSV đúng chỗ, copy prompt → paste vào Claude Code → nhận báo cáo trong vài phút.

🔧
Thực hành

Chọn một workflow bạn làm mỗi tuần hoặc mỗi tháng — dù là báo cáo, content, review leads, hay bất cứ thứ gì. Dành 20 phút viết ra các bước theo Bước 1 (Capture): inputs là gì, outputs là gì, bao nhiêu bước thủ công? Sau đó tạo prompt template cho bước mất thời gian nhất. Chạy thử với Claude Code. Đây là seed của workflow system của bạn.
