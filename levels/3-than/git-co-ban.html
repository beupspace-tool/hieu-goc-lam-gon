Bài 3.3: Git cho người vận hành

⏱ ~6 phút đọc Level 3 Git
💡
Nguyên lý

Git không phải dành riêng cho developer. Khi Claude Code chỉnh sửa hàng chục files trong một session, bạn cần biết: Claude đã thay đổi gì, thay đổi đó có đúng không, và nếu sai thì quay lại bằng cách nào. Git trả lời cả ba câu hỏi đó. Không có Git = không có safety net = rủi ro mất dữ liệu thật sự.

Git thực sự là gì

Hãy nghĩ về tính năng "Version History" trong Google Docs — bạn có thể xem ai đã sửa gì, lúc nào, và restore về phiên bản trước nếu cần. Git làm điều tương tự nhưng cho toàn bộ thư mục project, không chỉ một file.

Mỗi lần bạn "commit", Git chụp một snapshot toàn bộ trạng thái project tại thời điểm đó. Bạn có thể xem lịch sử đầy đủ, so sánh bất kỳ hai thời điểm nào, và quay lại bất kỳ snapshot nào — kể cả snapshot từ 6 tháng trước.

Với Claude Code, Git đặc biệt quan trọng vì:

  • Claude có thể sửa nhiều files cùng lúc trong vài giây — không có Git, bạn không biết Claude đã thay đổi gì
  • Claude đôi khi hiểu sai yêu cầu và làm sai — Git cho phép bạn undo toàn bộ session đó
  • Bạn cần record lại "Claude đã làm gì" để có thể reproduce hoặc audit sau này

5 lệnh Git cần biết

Lệnh Dùng để
git init Khởi tạo Git trong thư mục hiện tại. Chạy một lần khi bắt đầu project mới.
git add . Đánh dấu tất cả thay đổi để đưa vào snapshot tiếp theo. Dấu chấm nghĩa là "mọi thứ trong thư mục này".
git commit -m "mô tả" Tạo snapshot với một mô tả ngắn. Đây là thao tác quan trọng nhất — làm thường xuyên.
git log --oneline Xem lịch sử các snapshots, mỗi cái một dòng. Dễ đọc hơn git log thông thường.
git diff Xem những gì đã thay đổi kể từ snapshot cuối. Cực kỳ hữu ích sau khi Claude sửa files — bạn thấy chính xác Claude đã làm gì.

Workflow chuẩn khi làm việc với Claude Code

Quy trình đơn giản mà hiệu quả:

# 1. Trước khi bắt đầu session với Claude
git add . && git commit -m "before: claude session - mô tả task"

# 2. Claude làm việc, sửa files...

# 3. Sau khi Claude hoàn thành, xem Claude đã làm gì
git diff

# 4. Nếu ổn, commit kết quả
git add . && git commit -m "feat: claude added/fixed/updated..."

# 5. Nếu Claude làm sai, revert về snapshot trước
git checkout -- .   # Hoặc dùng: git restore .

Bước 1 và 2 là quan trọng nhất: commit trước khi giao task cho Claude. Đây là checkpoint để bạn rollback về nếu cần.

Đọc output của git diff

Sau khi Claude sửa files, chạy git diff. Bạn sẽ thấy output như này:

diff --git a/reports/monthly.md b/reports/monthly.md
--- a/reports/monthly.md
+++ b/reports/monthly.md
@@ -15,7 +15,9 @@
 ## Doanh thu tháng 3

-Tổng doanh thu: 450 triệu VND
+Tổng doanh thu: 487 triệu VND
+Tăng trưởng: +8.2% so với tháng 2
+Đạt 97% kế hoạch tháng

Dòng bắt đầu bằng - (đỏ) = những gì bị xóa. Dòng bắt đầu bằng + (xanh) = những gì được thêm vào. Trong ví dụ trên, Claude đã cập nhật số liệu doanh thu và thêm hai dòng phân tích mới.

GitHub — Git trên cloud

Git là công cụ local trên máy bạn. GitHub là dịch vụ lưu trữ Git repository trên cloud — backup + collaboration.

Để đẩy project lên GitHub:

# Tạo repo mới trên github.com, sau đó:
git remote add origin https://github.com/username/repo-name.git
git branch -M main
git push -u origin main

# Lần sau chỉ cần:
git push

Claude Code tích hợp sẵn với GitHub qua MCP (bài 3.2). Khi có GitHub MCP, Claude có thể tạo PR, review code, manage issues trực tiếp từ Claude Code session.

Branching — thử nghiệm an toàn

Khi muốn thử nghiệm một thay đổi lớn mà không ảnh hưởng project chính:

# Tạo branch mới và chuyển sang đó
git checkout -b thu-nghiem-tinh-nang-moi

# Làm việc, Claude sửa files...

# Nếu ổn, merge về main
git checkout main
git merge thu-nghiem-tinh-nang-moi

# Nếu không ổn, xóa branch đó đi
git checkout main
git branch -D thu-nghiem-tinh-nang-moi

Nguyên tắc: main = phiên bản đang chạy thực tế. Branch = khu vực thử nghiệm. Chỉ merge vào main khi đã verify kết quả ổn.

Git Worktrees — chạy song song (nâng cao)

Một tính năng ít người biết nhưng cực kỳ mạnh: Git Worktrees cho phép bạn mở nhiều working copies của cùng một repository, mỗi copy ở một branch khác nhau.

Ứng dụng thực tế: chạy 5-10 Claude Code sessions cùng lúc, mỗi session trên một worktree riêng, làm 5-10 task khác nhau song song. Không có conflict vì mỗi session làm việc trên folder riêng biệt hoàn toàn.

# Tạo worktree mới cho task song song
git worktree add ../project-task-marketing -b task-marketing
git worktree add ../project-task-report -b task-report

# Mỗi folder được mở trong Claude Code session riêng
# 30 phút sequential → 5 phút parallel

Worktrees là công cụ của power users — bạn chưa cần ngay, nhưng khi workflow của bạn scale lên, đây là bước nhảy vọt về năng suất.

🔧
Thực hành

Trong một thư mục test mới, chạy: git init && git add . && git commit -m "initial commit". Mở Claude Code trong thư mục đó, yêu cầu Claude tạo một file nào đó. Sau khi Claude hoàn thành, chạy git diff để xem Claude đã làm gì. Đây là workflow bạn sẽ dùng mỗi ngày.
