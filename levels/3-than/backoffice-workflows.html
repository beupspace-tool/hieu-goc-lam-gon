Bài 3.8: Backoffice workflows

⏱ ~6 phút đọc Level 3 Operations
💡
Nguyên lý

Backoffice — kế toán, HR, admin, compliance — chiếm 30-40% thời gian của nhiều doanh nghiệp SME. Đây là công việc quan trọng nhưng không tạo ra giá trị trực tiếp. Claude Code tốt nhất ở đây: structured data, repeatable processes, clear inputs và outputs. Mỗi giờ bạn automate backoffice = một giờ trả lại cho công việc có giá trị thực sự.

Những việc Claude Code làm tốt nhất trong backoffice

🧾 Xử lý hóa đơn và chứng từ -60% thời gian

Claude đọc invoice (PDF text hoặc CSV) → extract fields: nhà cung cấp, số tiền, ngày, hạng mục → structured data → update spreadsheet hoặc chuẩn bị data import vào phần mềm kế toán.

Đọc tất cả file invoice trong folder invoices/tháng-6/
Với mỗi invoice, extract:
- Tên nhà cung cấp
- Số hóa đơn
- Ngày phát hành
- Ngày đến hạn thanh toán
- Số tiền (VND)
- Mã loại chi phí (xem categories.md)
- Trạng thái: đã thanh toán / chưa thanh toán

Tổng hợp vào file invoice-summary-t6.csv
Flag invoices quá hạn hoặc > 50 triệu VND để review ưu tiên.
📊 Report generation tự động -75% thời gian

Không cần mở Excel, viết formula, format lại. Claude nhận raw data, phân tích, tạo formatted report với executive summary, charts data, và recommendations.

Đọc file sales-data-q2.csv
Tạo quarterly report bao gồm:
1. Executive summary (5 bullet points, tối đa 30 từ/bullet)
2. Doanh thu theo tháng: tháng 4, 5, 6 (so sánh % vs Q1)
3. Top 10 sản phẩm theo doanh thu
4. Top 5 khách hàng theo giá trị
5. Phân tích vùng địa lý nếu có data
6. 3 điểm đáng chú ý và đề xuất hành động

Lưu vào reports/Q2-2026.md với format chuẩn

✅ Compliance và kiểm tra chứng từ -40% thời gian

Kiểm tra tài liệu có đủ thông tin bắt buộc không. Hữu ích cho hợp đồng, báo cáo định kỳ, hồ sơ thuế, giấy phép.

Đọc tất cả hợp đồng trong folder contracts/q2/
Với mỗi hợp đồng, kiểm tra checklist:
- [ ] Đầy đủ thông tin hai bên (tên, địa chỉ, mã số thuế)
- [ ] Giá trị hợp đồng và điều khoản thanh toán rõ ràng
- [ ] Ngày hiệu lực và ngày kết thúc
- [ ] Điều khoản chấm dứt hợp đồng
- [ ] Chữ ký và con dấu (ghi "cần verify thủ công")
- [ ] Phụ lục đính kèm nếu được tham chiếu trong nội dung

Flag hợp đồng thiếu bất kỳ mục nào trong checklist.
📋 Data entry và cleanup -90% thời gian

Dọn dẹp data lộn xộn, standardize format, detect duplicates, fill missing values. Việc này thường mất hàng giờ với Excel — Claude làm trong vài phút.

File customer-data.csv có các vấn đề sau:
- Tên công ty: lúc viết tắt, lúc viết đầy đủ (VD: "CTY TNHH" vs "Công ty TNHH")
- Số điện thoại: nhiều format khác nhau (không có 0, có dấu -, có khoảng cách)
- Email: một số bị sai format
- Tỉnh thành: không nhất quán (HCM, Hồ Chí Minh, TPHCM, TP. HCM)

Làm sạch toàn bộ file:
1. Standardize tên công ty
2. Format SĐT về: 0xxx-xxx-xxx
3. Flag emails sai format (không sửa — chỉ flag)
4. Standardize tên tỉnh theo list trong provinces.csv
5. Tạo cleaned_customer-data.csv + report về những gì đã thay đổi

Lưu ý bảo mật với data nhạy cảm

Backoffice data thường chứa thông tin nhạy cảm: lương, thông tin khách hàng, dữ liệu tài chính. Một số nguyên tắc:

  • Claude Code chạy local — data không gửi đến cloud (trừ phần Claude API process). Tuy nhiên, với sensitive data, hãy review Anthropic Privacy Policy.
  • Anonymize trước khi test — khi đang build workflow mới, dùng sample data thay vì real customer data.
  • Không commit data vào git — thêm data/ và *.csv vào .gitignore cho projects xử lý sensitive data.
  • Access control — nếu dùng Claude Code trong team context, đảm bảo chỉ người có quyền mới có thể chạy workflows với data nhất định.
🔧
Thực hành

Tạo file invoices.csv với 10 hóa đơn mẫu (dữ liệu giả): tên nhà cung cấp, số tiền, ngày, hạng mục. Yêu cầu Claude: (1) Tóm tắt tổng chi theo hạng mục, (2) Nhóm theo nhà cung cấp và tính trung bình, (3) Flag hóa đơn có số tiền lớn hơn 10 triệu VND, (4) Tạo báo cáo chi tiêu tháng với executive summary. Đo thời gian. Sau đó thử với data thật của tháng gần nhất và cảm nhận sự khác biệt.
