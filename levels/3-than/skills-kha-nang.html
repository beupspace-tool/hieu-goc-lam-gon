Bài 3.4: Skills — Khả năng mở rộng

⏱ ~7 phút đọc Level 3 Skills
💡
Nguyên lý

Skills là cách bạn dạy Claude Code làm đúng việc, đúng cách, mỗi lần đều nhất quán — không cần nhắc lại từ đầu. Mỗi skill là một bộ hướng dẫn chuyên biệt: Claude đọc nó khi task match, sau đó thực thi theo quy trình được định nghĩa sẵn. Thay vì bạn prompting mỗi lần, skill prompts thay bạn.

Skill là gì — định nghĩa đơn giản

Một skill = một thư mục chứa hướng dẫn cho Claude làm một việc cụ thể. Khi bạn gõ /plan, Claude đọc skill "plan", hiểu rằng task này cần approach như thế nào, và thực thi theo.

Skills giải quyết một vấn đề quan trọng: Claude rất giỏi, nhưng không biết cách làm việc theo quy trình của bạn trừ khi bạn định nghĩa quy trình đó. Skills là nơi bạn viết ra "khi Claude làm X, tôi muốn Claude làm theo cách Y, với format Z, không bao giờ làm W".

Built-in skills trong Claude Code

Claude Code đi kèm với một số skills built-in — dùng được ngay mà không cần cài gì thêm:

/plan

Tạo implementation plan trước khi bắt tay làm. Phân tích task, chia phase, liệt kê file sẽ thay đổi. Không thực thi — chỉ lập kế hoạch để bạn review.

/test

Chạy tests và phân tích kết quả. Báo cáo what passed, what failed, và suggest fixes.

/debug

Debug có cấu trúc: reproduce lỗi → trace root cause → propose fix → implement → verify. Không đoán mò.

/fix

Fix một lỗi cụ thể. Khác với /debug ở chỗ bạn đã biết lỗi là gì, chỉ cần sửa.

/docs

Tạo documentation cho code hoặc project. Tự động extract từ code và tạo docs có cấu trúc.

/review

Code review với nhận xét cụ thể về quality, security, performance. Trả về danh sách actionable items.

Community skills — 1000+ lựa chọn

Cộng đồng đã build một kho skills lớn. Một số nguồn đáng chú ý:

awesome-claude-code-toolkit (rohitg00 trên GitHub): 135+ agent definitions, 42 slash commands. Có các skills cho SEO, marketing, data analysis, customer support, và nhiều hơn nữa.

Claude Code Community trên Discord và Reddit — nơi developers chia sẻ skills họ tự build cho workflows cụ thể.

Ví dụ một số community skills phổ biến cho business:

  • SEO skill: Audit website, keyword research, optimize meta tags, generate schema markup
  • Marketing skill: Tạo content calendar, viết copy theo brand voice, adapt content cho từng platform
  • Database skill: Query optimization, report generation, data cleanup
  • HR skill: Phân tích JD, screening criteria, candidate evaluation rubric
  • Finance skill: Phân tích báo cáo tài chính, budget variance analysis

Anatomy của một skill

Một skill được lưu trong ~/.claude/skills/{skill-name}/ với cấu trúc:

~/.claude/skills/seo-audit/
  ├── SKILL.md   ← hướng dẫn chính cho Claude
  ├── scripts/  ← bash/python scripts tự động hóa
  │   └── run-audit.sh
  ├── references/ ← data tham chiếu
  │   └── seo-checklist.md
  └── templates/ ← output templates
      └── audit-report.md

SKILL.md là file quan trọng nhất — định nghĩa khi nào skill này được kích hoạt, Claude nên làm gì, bước nào theo bước nào, và output format mong muốn là gì.

Ví dụ đầu của một SKILL.md đơn giản:

# SEO Audit Skill

## Kích hoạt khi
- User dùng `/seo` hoặc `/seo-audit`
- User hỏi về SEO của website

## Quy trình
1. Đọc file sitemap.xml nếu có
2. Kiểm tra meta tags trong HTML files
3. Phân tích heading structure (H1, H2, H3)
4. Check image alt texts
5. Tạo báo cáo theo template trong templates/audit-report.md

## Không bao giờ
- Không tự sửa file mà chưa có approval
- Không suggest keywords mà không có research data

Cách install community skill

# Cách 1: Clone trực tiếp vào skills folder
git clone https://github.com/user/skill-name ~/.claude/skills/skill-name

# Cách 2: Copy thủ công
mkdir -p ~/.claude/skills/my-skill
# Copy SKILL.md và các files liên quan vào đó

# Verify skill đã available
# Restart Claude Code → gõ /skill-name để test

Skills lưu trong ~/.claude/skills/ available cho tất cả projects. Muốn skill chỉ dùng cho một project, lưu trong {project}/.claude/skills/.

Tự tạo skill cho workflow riêng

Đây là lúc skills trở nên thực sự mạnh. Bạn có quy trình riêng — weekly report, content review, khách hàng onboarding — bạn có thể đóng gói thành skill để Claude thực thi nhất quán mỗi lần.

Nguyên tắc khi viết skill:

  • Cụ thể về steps, không mơ hồ ("viết summary" → "viết executive summary 3 bullet points, tối đa 30 từ mỗi bullet")
  • Định nghĩa rõ output format — Claude nên trả về gì, lưu ở đâu, tên file ra sao
  • Ghi rõ "không bao giờ làm gì" — ranh giới quan trọng không kém steps
  • Thêm ví dụ input/output tốt vào SKILL.md — Claude học từ examples
🔧
Thực hành

Thử /plan ngay trong Claude Code session hiện tại. Gõ một request phức tạp — ví dụ: "Tôi muốn tạo hệ thống theo dõi leads từ website, lưu vào CSV, tự động gửi email follow-up sau 24h." Thêm /plan vào đầu. Quan sát cách Claude tạo plan có cấu trúc thay vì bắt tay làm ngay. Approve plan rồi mới cho Claude execute. So sánh kết quả với khi không dùng /plan.
