Bài 3.1: MCP — Nguyên lý

⏱ ~7 phút đọc Level 3 MCP
💡
Nguyên lý

MCP biến Claude Code từ tool đọc file thành platform kết nối mọi thứ. Trước MCP, Claude chỉ làm việc với file local trên máy bạn. Sau MCP, Claude có thể đọc task từ Notion, tạo PR trên GitHub, gửi message trên Slack, query database — tất cả trong cùng một session. Hiểu MCP = hiểu tại sao Claude Code mạnh hơn bất kỳ chatbot nào.

Vấn đề MCP giải quyết

Hãy tưởng tượng bạn có một nhân viên xuất sắc, nhưng nhân viên đó chỉ có thể làm việc trong một căn phòng kín — không email, không điện thoại, không kết nối với hệ thống nào của công ty. Tài năng đến đâu cũng bị giới hạn bởi thông tin họ có thể tiếp cận.

Đó chính xác là tình trạng của Claude Code trước khi có MCP. Claude Code mặc định chỉ đọc và viết file trên máy tính của bạn. Muốn Claude biết nội dung Notion workspace? Bạn phải copy-paste thủ công. Muốn Claude tạo issue trên GitHub? Phải mở browser, dùng tay. Muốn Claude kiểm tra dữ liệu từ database? Bạn phải export ra file CSV trước.

MCP (Model Context Protocol) phá vỡ bức tường đó. Đây là chuẩn giao tiếp mở cho phép Claude Code kết nối trực tiếp với bất kỳ dịch vụ nào có MCP server — mà không cần bạn làm trung gian.

Kiến trúc MCP: ba thành phần

┌─────────────────────────────────────────────────────────┐
│                    Claude Code (Client)                  │
│   "Liệt kê tất cả tasks trong Notion workspace của tôi" │
└────────────────────┬────────────────────────────────────┘
                     │ gọi tool
                     ▼
┌─────────────────────────────────────────────────────────┐
│                   MCP Server                            │
│   @notionhq/mcp-server  ←→  Notion API                  │
│   @github/mcp-server    ←→  GitHub API                  │
│   postgres-mcp          ←→  PostgreSQL                  │
└─────────────────────────────────────────────────────────┘
                     │ trả kết quả
                     ▼
┌─────────────────────────────────────────────────────────┐
│                    Claude Code                           │
│   Nhận data → phân tích → trả lời → thực hiện action   │
└─────────────────────────────────────────────────────────┘

MCP Client là Claude Code — phía "yêu cầu". Claude biết các tools nào đang available và gọi chúng khi cần.

MCP Server là cầu nối — chạy local trên máy bạn, expose một tập tools cụ thể cho Claude. Ví dụ: Notion MCP server expose các tools như list_pages, create_page, update_block. GitHub MCP server expose create_pull_request, list_issues, review_code.

External Service là Notion, GitHub, Slack, database của bạn — phía "thực thi". MCP server dùng API của service này để thực sự làm việc.

Điểm quan trọng: MCP server chạy local, trên máy của bạn. Data không đi qua third party nào. Claude Code giao tiếp với MCP server qua localhost, không qua internet.

MCP là chuẩn mở — không phải lock-in

Anthropic publish MCP spec vào cuối 2024 dưới dạng open standard. Quyết định này thay đổi mọi thứ: thay vì Anthropic tự build từng integration, cả cộng đồng — từ startup nhỏ đến enterprise lớn — có thể build MCP server cho dịch vụ của họ.

Kết quả đến năm 2026: 200+ MCP servers available, từ Notion, Slack, Linear đến Stripe, Salesforce, PostgreSQL, MongoDB, Google Sheets. Và con số này tăng mỗi tuần.

Quan trọng hơn: OpenAI, Google (Gemini), Microsoft (Copilot Studio), và Amazon (Bedrock) đều đã adopt MCP spec. Điều này có nghĩa là skill bạn học với Claude Code hôm nay — cách cấu hình MCP, cách thiết kế workflow — sẽ chuyển được sang các AI platform khác. Không bị lock-in vào một vendor.

So sánh dễ hiểu: MCP giống USB

Trước khi có USB, mỗi thiết bị ngoại vi — chuột, keyboard, printer, camera — đều có cổng kết nối riêng. Mua printer mới phải kiểm tra máy tính có đúng cổng không. Mua camera phải có driver riêng.

USB đến và giải quyết hết: 1 cổng chuẩn, tất cả thiết bị đều dùng. Bạn không cần biết printer của bạn kết nối với máy tính như thế nào — chỉ cần cắm USB là xong.

MCP là USB của thế giới AI. Trước MCP, mỗi AI tool tự build integration riêng với từng service — Copilot tích hợp GitHub theo cách riêng, ChatGPT tích hợp Zapier theo cách riêng, mỗi tool cần adapter riêng biệt. MCP = 1 chuẩn cho tất cả.

Ví dụ business thực tế

📋

Notion MCP

Claude đọc task list từ Notion → tạo báo cáo tiến độ weekly. Hoặc ngược lại: Claude tạo pages Notion từ brief của bạn.

🐙

GitHub MCP

Claude tạo PR, review code, manage issues, tag releases — tất cả bằng ngôn ngữ tự nhiên thay vì nhớ lệnh git.

🗄️

Database MCP

Claude query PostgreSQL trực tiếp — "Doanh thu tháng này so với tháng trước" thay vì bạn phải viết SQL query.

💬

Slack MCP

Claude gửi notification vào đúng channel khi task hoàn thành. Hoặc đọc messages để tóm tắt cuộc thảo luận team.

📊

Google Sheets MCP

Claude đọc spreadsheet thực tế → phân tích → ghi kết quả vào sheet mới. Không cần export CSV thủ công.

📧

Email MCP

Claude tìm kiếm email theo chủ đề, tóm tắt threads dài, soạn reply — trực tiếp từ inbox của bạn.

Tại sao điều này quan trọng với doanh nghiệp bạn

Hầu hết doanh nghiệp SME đang vận hành trên 5-15 phần mềm khác nhau: CRM, kế toán, project management, email, chat, file storage. Mỗi phần mềm có dữ liệu riêng, không nói chuyện với nhau. Nhân viên mất 30-40% thời gian copy-paste dữ liệu giữa các hệ thống.

MCP không thay thế các phần mềm đó — nhưng nó đặt Claude Code làm lớp điều phối ở trên. Claude có thể đọc dữ liệu từ bất kỳ nguồn nào, xử lý, và ghi kết quả vào bất kỳ đâu — trong một workflow liên tục, không cần bạn làm trung gian.

Ví dụ thực tế: "Đọc danh sách leads mới từ CRM → tìm thông tin công ty trên web → tạo personalized email cho từng lead → lưu draft vào Gmail → tạo task follow-up trong Notion." Trước MCP: quy trình này mất 2-3 giờ mỗi tuần. Với MCP: 1 lệnh, 5 phút.

Giới hạn cần biết

MCP không phải magic. Một số điều cần hiểu đúng:

Phụ thuộc vào MCP server quality: Chất lượng tích hợp tùy thuộc vào người build MCP server. Server do công ty chính thức build (Notion, Stripe) thường ổn định hơn community builds.

Rate limits: Mỗi service có rate limit API riêng. Nếu bạn dùng Claude Code với Notion MCP để xử lý 1000 pages cùng lúc, bạn có thể bị block tạm thời bởi Notion API.

Permission scope: MCP server chỉ có quyền làm những gì API key của bạn cho phép. Hãy tạo API key với minimal permissions cần thiết — không bao giờ dùng "admin key" cho MCP.

🔧
Thực hành

Xem danh sách MCP servers tại github.com/modelcontextprotocol/servers. Chọn 3 tool bạn dùng hàng ngày nhất (Notion, Slack, email, CRM, v.v.) và kiểm tra xem có MCP server chính thức không. Nếu có, ghi chú lại — bạn sẽ setup chúng ở bài tiếp theo.

⚠️
Lưu ý quan trọng

MCP server chạy local trên máy bạn — không phải trên server của Anthropic hay bất kỳ bên thứ ba nào. Điều này tốt cho privacy. Nhưng nó cũng có nghĩa là bạn cần cài đặt Node.js (cho hầu hết MCP servers) và maintain các dependencies. Bài 3.2 sẽ hướng dẫn chi tiết.
