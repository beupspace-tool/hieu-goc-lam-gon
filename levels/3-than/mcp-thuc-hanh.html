Bài 3.2: MCP thực hành

⏱ ~8 phút đọc Level 3 Setup
💡
Nguyên lý

Setup MCP đúng cách lần đầu tiết kiệm hàng giờ debug về sau. Ba điều quan trọng nhất: (1) file cấu hình đúng vị trí, (2) API keys không commit lên git, (3) biết cách kiểm tra xem tools có available không. Mọi thứ khác đều có thể fix sau.

File cấu hình MCP

MCP được cấu hình qua file JSON. Claude Code đọc file này khi khởi động và biết những servers nào cần connect. Bạn có hai lựa chọn vị trí:

Project-level — file .mcp.json trong thư mục project:

your-project/
├── .mcp.json          ← MCP config cho project này
├── CLAUDE.md
└── ...

Dùng khi bạn muốn MCP config riêng cho từng project. Ví dụ: project marketing dùng Notion + Buffer MCP, project dev dùng GitHub + database MCP.

Global — file ~/.claude/settings.json trong home directory. Dùng khi muốn MCP available cho mọi project.

Format file cấu hình:

{
  "mcpServers": {
    "ten-server": {
      "command": "npx",
      "args": ["-y", "@package/mcp-server"],
      "env": {
        "API_KEY": "your-key-here"
      }
    }
  }
}

Setup Notion MCP — từng bước

Notion là một trong những MCP servers được dùng nhiều nhất. Đây là cách setup từ zero:

  1. Tạo Notion Integration

    Vào notion.so/my-integrations → click "New integration" → đặt tên (ví dụ: "Claude Code") → chọn workspace → set capabilities: Read content, Update content, Insert content.

  2. Lấy Integration Token

    Sau khi tạo, bạn thấy "Internal Integration Token" bắt đầu bằng secret_. Copy token này — đây là API key của bạn.

  3. Share pages với Integration

    Mặc định, integration không có quyền đọc page nào. Vào từng Notion page bạn muốn Claude truy cập → click "..." → "Connections" → chọn integration vừa tạo. Hoặc share toàn bộ workspace nếu bạn muốn Claude đọc tất cả.

  4. Tạo file .mcp.json

    Trong thư mục project, tạo file .mcp.json với nội dung sau:

    {
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/mcp-server"],
      "env": {
        "NOTION_TOKEN": "secret_xxx_token_của_bạn"
      }
    }
  }
}

    Thay secret_xxx_token_của_bạn bằng token thật từ bước 2.

  5. Thêm .mcp.json vào .gitignore ngay lập tức

    File này chứa API key — không bao giờ được commit lên git.

    # Thêm vào .gitignore
.mcp.json
.claude/settings.local.json
  6. Restart Claude Code và kiểm tra

    Đóng và mở lại Claude Code. Gõ /mcp để xem danh sách MCP servers đang active. Bạn sẽ thấy "notion" trong danh sách với các tools available.

  7. Test thực tế

    Hỏi Claude: "Liệt kê tất cả pages trong Notion workspace của tôi" hoặc "Tìm page có tiêu đề chứa từ 'marketing'." Nếu Claude trả về kết quả thực, setup thành công.

Setup GitHub MCP

GitHub MCP cho phép Claude tạo PR, review code, manage issues bằng ngôn ngữ tự nhiên.

Trước tiên, authenticate với GitHub CLI nếu chưa làm:

# Cài GitHub CLI nếu chưa có
brew install gh   # macOS
# hoặc: winget install GitHub.cli   # Windows

# Authenticate
gh auth login

Thêm GitHub MCP vào .mcp.json:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token_here"
      }
    }
  }
}

Tạo Personal Access Token tại github.com/settings/tokens → "Generate new token (classic)" → check các scopes: repo, read:org. Chỉ cần đúng permissions — không cần "admin:repo" nếu bạn chỉ muốn đọc/tạo PR.

Setup nhiều servers cùng lúc

File .mcp.json cho phép cấu hình nhiều servers trong cùng object mcpServers:

{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/mcp-server"],
      "env": { "NOTION_TOKEN": "secret_xxx" }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxx" }
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Documents",
        "/Users/yourname/Desktop"
      ]
    }
  }
}

Filesystem MCP server cho phép Claude đọc/ghi file ngoài thư mục project hiện tại — hữu ích khi bạn muốn Claude truy cập tài liệu ở nhiều vị trí khác nhau.

Debugging khi có lỗi

MCP setup thất bại thường do một trong ba nguyên nhân:

1. Node.js chưa cài hoặc version cũ. Hầu hết MCP servers yêu cầu Node.js 18+. Kiểm tra: node --version. Nếu chưa có, tải tại nodejs.org.

2. API key sai hoặc thiếu permissions. Kiểm tra lại token trong .mcp.json. Thử gọi API trực tiếp bằng curl để verify key còn hiệu lực.

3. File cấu hình sai cú pháp JSON. Dùng jsonlint.com để validate JSON trước khi restart Claude Code.

Xem log lỗi MCP: trong Claude Code, gõ /mcp → xem trạng thái từng server. Server có lỗi sẽ hiển thị error message cụ thể.

Bảo mật API keys

Nguyên tắc cốt lõi: API keys không bao giờ được commit lên git repository, dù là private repo.

Cách tốt nhất: lưu API keys trong environment variables của hệ điều hành, sau đó tham chiếu trong .mcp.json:

# Thêm vào ~/.zshrc hoặc ~/.bashrc
export NOTION_TOKEN="secret_xxx"
export GITHUB_TOKEN="ghp_xxx"
# Trong .mcp.json, dùng process.env
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/mcp-server"],
      "env": { "NOTION_TOKEN": "${NOTION_TOKEN}" }
    }
  }
}

Hoặc dùng file .env (cũng phải add vào .gitignore) để lưu secrets theo project.

⚠️
Không commit secrets

KHÔNG commit file .mcp.json chứa API keys lên GitHub — dù repo đó là private. Private repos có thể bị leak qua nhiều cách: fork, collaborators, GitHub Actions logs. Luôn thêm vào .gitignore ngay khi tạo file. Nếu lỡ commit, revoke token ngay lập tức và tạo token mới.

🔧
Thực hành

Setup Notion MCP theo 7 bước trên. Sau khi xong, hỏi Claude: "Liệt kê tất cả pages trong Notion workspace của tôi và cho biết page nào được sửa gần nhất." Nếu Claude trả về danh sách thực từ Notion của bạn — bạn vừa mở khóa khả năng quan trọng nhất của Level 3. Ghi lại thời gian setup — thường 15-20 phút lần đầu, 5 phút cho lần sau.
