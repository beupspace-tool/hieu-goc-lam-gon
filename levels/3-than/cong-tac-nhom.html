Bài 3.6: Cộng tác nhóm

⏱ ~6 phút đọc Level 3 Team
💡
Nguyên lý

Khi cả team dùng Claude Code, vấn đề không phải là kỹ thuật — mà là nhất quán. Mỗi người một cách dùng khác nhau → output chất lượng không đồng đều → team lead phải fix nhiều hơn làm. Shared CLAUDE.md và conventions chung giải quyết vấn đề này: AI của mọi người trong team hoạt động theo cùng một "playbook".

Shared CLAUDE.md — nền tảng của team

CLAUDE.md của team là file quan trọng nhất khi nhiều người cùng dùng Claude Code cho một project. Nó được lưu trong repo (commit và sync qua git), và mỗi thành viên team đều dùng khi làm việc trên project đó.

Những gì cần có trong shared CLAUDE.md của team:

# Team CLAUDE.md — Marketing Team

## Project Context
Chúng ta là team marketing của XYZ Corp. Claude Code hỗ trợ content creation,
data analysis, và workflow automation. Khách hàng mục tiêu: B2B SME tại Việt Nam.

## Brand Voice
- Tông: professional nhưng không quá formal
- Ngôn ngữ: tiếng Việt, có thể mix một số thuật ngữ tiếng Anh khi cần
- Không dùng: quá nhiều emoji, câu hỏi tu từ kiểu "Bạn đã biết chưa?"

## Tech Stack & Tools
- Content calendar: Notion (xem MCP config)
- Analytics: Google Analytics 4
- Email: Mailchimp
- Social scheduling: Buffer

## File Conventions
- Blog drafts: content/drafts/YYYY-MM-DD-slug.md
- Social posts: social/YYYY-WW/ (theo tuần)
- Reports: reports/YYYY-MM/ (theo tháng)

## Commit Convention
- feat: content mới
- fix: sửa lỗi
- refactor: cải thiện không thêm content mới
- docs: cập nhật documentation

## Không làm
- Không publish trực tiếp lên website mà chưa có approval
- Không xóa file trong reports/ mà không backup
- Không commit file chứa customer data

Shared vs. private — phân tách đúng cách

Shared — commit vào git

  • CLAUDE.md với conventions
  • .mcp.json (không chứa secrets)
  • Prompt templates trong templates/
  • Workflow documentation
  • Skills trong .claude/skills/

Private — không commit

  • API keys (Notion token, GitHub token)
  • .claude/settings.local.json
  • .env files với credentials
  • Personal preferences
  • Draft content chưa review

Cách tách secrets: file .mcp.json shared chứa server config không có credentials. Mỗi người tự lưu credentials trong .claude/settings.local.json hoặc environment variables của máy họ.

# .mcp.json — SAFE to commit (no secrets)
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/mcp-server"],
      "env": {
        "NOTION_TOKEN": "${NOTION_TOKEN}"
      }
    }
  }
}

# Mỗi người tự set trong ~/.zshrc:
export NOTION_TOKEN="secret_của_tôi_riêng"

Claude Code Team plan

Anthropic cung cấp Team plan ($25+/người/tháng) với các tính năng dành riêng cho tổ chức:

  • Admin dashboard: xem usage của từng thành viên, quản lý seat
  • Usage monitoring: track token consumption per user/project
  • Centralized billing: một hóa đơn cho toàn team, phân bổ chi phí theo phòng ban
  • SSO (Single Sign-On): tích hợp với Google Workspace hoặc Okta
  • Priority support: response time ưu tiên

Với team dưới 5 người, Pro plan cá nhân cho từng người thường cost-effective hơn Team plan. Team plan worth it từ 5-6 người trở lên khi admin overhead và billing consolidation trở nên quan trọng.

Git branching strategy cho team

Khi nhiều người dùng Claude Code trên cùng project, cần quy tắc rõ ràng để tránh conflict:

# Mỗi thành viên làm việc trên branch riêng
git checkout -b feature/nguyen-content-june
git checkout -b feature/linh-data-cleanup
git checkout -b feature/tuan-template-update

# Khi xong, tạo Pull Request để review
gh pr create --title "Content batch June week 1" --base main

# Lead review và merge, không tự merge

Quy tắc quan trọng: không ai trực tiếp commit lên main. Mọi thay đổi qua Pull Request, dù nhỏ. Điều này cho phép code review và tránh tình huống hai người vô tình ghi đè lên nhau.

Agent Teams — điều phối task lớn

Agent Teams là tính năng nâng cao cho phép một "lead agent" phân chia task lớn thành subtasks và giao cho nhiều "worker agents" chạy song song.

Ví dụ thực tế: "Audit toàn bộ 200 trang web của tôi về SEO" là task quá lớn cho một Claude session. Với Agent Teams:

  • Lead agent phân tích sitemap, chia thành 10 nhóm 20 trang
  • 10 worker agents chạy song song, mỗi agent audit 20 trang
  • Lead agent tổng hợp kết quả, prioritize issues, tạo final report

Thời gian: sequential 4 giờ → parallel 30 phút. Agent Teams còn ở giai đoạn early access — nhưng đây là hướng mà team workflows sẽ đi trong tương lai gần.

Code review bằng Claude Code

Workflow thực tế cho team dùng Claude Code review lẫn nhau:

# Sau khi thành viên tạo PR:
# 1. Đọc diff của PR
git diff main...feature/nguyen-update

# 2. Yêu cầu Claude review
"Review những thay đổi trong diff này.
Tập trung vào: (1) logic errors, (2) security issues, (3) inconsistencies với CLAUDE.md.
Trả về list cụ thể, mỗi issue 1 dòng với file:line reference."

# 3. Claude output review comments
# 4. Human approves/rejects suggestions
# 5. Merge nếu clean

Claude review nhanh và không bỏ sót chi tiết. Human review đảm bảo Claude không bỏ qua context business. Kết hợp hai lớp review = chất lượng cao với tốc độ nhanh.

🔧
Thực hành

Tạo shared CLAUDE.md cho team bạn. Bao gồm tối thiểu: (1) mô tả project và khách hàng mục tiêu, (2) brand voice và ngôn ngữ, (3) file naming conventions, (4) commit message format, (5) danh sách những việc Claude không được làm mà không có approval. Commit file này lên shared repo và gửi cho team review. Đây là bước đầu tiên để team dùng Claude nhất quán.
