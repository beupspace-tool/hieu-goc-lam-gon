Bài 4.5: Prompt nâng cao

⏱ ~7 phút đọc Level 4 Cost optimization
💡
Nguyên lý

Prompt engineering cơ bản là viết prompt rõ hơn. Context engineering — cấp độ tiếp theo — là quản lý TOÀN BỘ thông tin Claude nhận được: CLAUDE.md, conversation history, file context, MCP data. Người giỏi context engineering ra output tốt hơn với chi phí ít hơn.

Context engineering: Toàn bộ picture

Khi Claude nhận một request, nó xử lý nhiều luồng context đồng thời:

  • System context: CLAUDE.md của bạn — instructions, rules, brand voice, workflows
  • Conversation history: Những gì đã nói trong session hiện tại
  • File context: Files Claude đã đọc trong session — source code, docs, data
  • MCP context: Data từ MCP servers — database queries, API responses
  • Tool outputs: Kết quả từ Bash, Read, Grep commands

Context engineering là biết cái nào nên có mặt, cái nào không, và khi nào. Context quá nhiều = Claude "bị loãng", không focus. Context thiếu = Claude đoán mò, ra output sai.

Nguyên tắc vàng: CLAUDE.md chứa những gì luôn cần (rules, brand, workflows). Conversation chứa những gì liên quan đến task hiện tại. Không dump tất cả vào một nơi.

Extended thinking / Adaptive thinking

Claude Opus 4 và Sonnet 4 có adaptive thinking — model tự quyết định khi nào cần "suy nghĩ" lâu hơn cho complex tasks. Bạn không cần config gì, nó tự kích hoạt.

Khi nào extended thinking thực sự hữu ích:

  • Multi-step reasoning: "Phân tích chiến lược pricing của 5 đối thủ và đề xuất cho công ty chúng tôi"
  • Complex code architecture: "Design database schema cho hệ thống inventory multi-warehouse"
  • Decision analysis: "So sánh 3 tech stack options với trade-offs cụ thể cho team 5 người"

Không cần cho: write email, format data, simple code tasks. Extended thinking tốn thêm tokens — dùng đúng chỗ.

Chain-of-thought prompting

Yêu cầu Claude giải thích reasoning trước khi đưa ra kết quả:

# Không tốt:
"Chiến lược marketing nào phù hợp cho startup SaaS B2B?"

# Tốt hơn với chain-of-thought:
"Phân tích từng bước: (1) Đặc điểm của startup SaaS B2B trong giai đoạn early-stage,
(2) Budget constraints thường gặp, (3) Channels nào có ROI cao nhất ở giai đoạn này,
(4) Đề xuất 3 chiến lược ưu tiên với justification cho mỗi cái."

Chain-of-thought tốn thêm ~30% tokens nhưng accuracy cao hơn rõ rệt với complex problems. Dùng khi kết quả sai có cost cao (business decisions, architecture choices).

Few-shot examples: Format nhất quán

Cho Claude 2-3 examples trước khi yêu cầu output — kết quả nhất quán hơn nhiều so với chỉ mô tả format:

# Thay vì:
"Viết product description ngắn gọn, professional, theo tone BEUP"

# Dùng few-shot:
"Viết product description theo format sau:

Ví dụ 1:
Product: Claude Code Pro Plan
Output: Dành cho team cần AI coding assistant mạnh nhất.
Không giới hạn context, không giới hạn models, không giới hạn tốc độ.

Ví dụ 2:
Product: Khóa học Claude Code
Output: Từ zero đến power user trong 5 levels.
Practical, không lý thuyết, kết quả ngay từ bài đầu tiên.

Bây giờ viết cho: [your product]"

Few-shot đặc biệt hiệu quả cho: social media posts, email subjects, product copy — bất kỳ thứ gì cần tone và format nhất quán.

Model routing: Chọn đúng model cho đúng task

Model Giá (input) Dùng cho Switch command
Haiku 3.5 ~$0.80/1M Format data, simple queries, classification, translation /model haiku
Sonnet 4 ~$3/1M Code generation, analysis, content writing — daily driver /model sonnet
Opus 4 ~$15/1M Complex reasoning, architecture design, strategic analysis /model opus

Trong thực tế: 80% tasks có thể dùng Sonnet. 15% simple tasks có thể dùng Haiku và tiết kiệm 75% chi phí. 5% tasks thực sự phức tạp xứng đáng Opus.

Routing strategy thực tế: Dùng Sonnet làm default. Khi task clearly simple (formatting, translation), switch sang Haiku. Khi gặp vấn đề quan trọng cần phân tích sâu (pricing strategy, architecture decision), switch sang Opus rồi về Sonnet.

Prompt caching: 90% cost reduction

Claude cache context được lặp lại — CLAUDE.md, large codebases, reference documents. Khi cache hit, chi phí giảm 90%.

Cách hoạt động: Lần đầu Claude đọc file 50KB, Anthropic charge full price. Lần thứ 2 trong 5 phút, cache hit — charge 10% của giá input. Cache TTL ~5 phút, sau đó expired.

Tối ưu cho caching:

  • Đặt content ít thay đổi (CLAUDE.md, docs) ở đầu conversation — dễ cache hơn
  • Làm việc trong session dài thay vì nhiều sessions ngắn — cache stay warm
  • Khi làm việc với codebase lớn, đọc files quan trọng lúc đầu session — Claude sẽ cache chúng

System prompts qua CLAUDE.md

CLAUDE.md là system prompt persistent của bạn — đặt rules ở đây override mọi thứ trong conversation. Cấu trúc CLAUDE.md hiệu quả:

# CLAUDE.md

## Role
Bạn là Senior Marketing Strategist cho BEUP Learning.
Luôn trả lời bằng tiếng Việt, tone professional nhưng thân thiện.

## Brand Voice
- Direct, không vòng vo
- Practical over theoretical
- Dùng ví dụ Việt Nam, không Western examples

## Workflow Rules
- Trước khi implement, luôn outline plan
- Sau mỗi task, hỏi "Tiếp theo bạn muốn làm gì?"
- Không suggest thêm features trừ khi được hỏi

## File Ownership
- Marketing content: content/*
- Analytics: data/*
- Never touch: .env, credentials/

Anti-patterns cần tránh

Vague creative requests: "Hãy sáng tạo về cách marketing sản phẩm này" — Claude không biết "sáng tạo" nghĩa là gì với bạn. Thay bằng: "Đề xuất 3 angle marketing khác nhau: 1 emotional, 1 data-driven, 1 social proof."

Task chaining quá dài trong 1 prompt: "Tìm hiểu competitors, viết strategy, tạo content calendar 3 tháng, viết 10 blog post outlines, tạo email sequence, và design social media plan." Claude sẽ làm tất cả nhưng không cái nào thực sự tốt. Break thành nhiều prompts riêng.

Bỏ qua error output: Khi Claude báo lỗi hoặc nói "tôi không chắc", đừng ignore. Đó là signal để clarify hoặc cung cấp thêm context — không phải để nói "cứ thử đi".

🔧
Thực hành

Thử 3 model tiers cho cùng 1 task — ví dụ "Đề xuất 3 headline cho landing page khóa học Claude Code": (1) /model haiku → run → note chất lượng và tốc độ. (2) /model sonnet → same task. (3) /model opus → same task. So sánh kết quả và quyết định tier nào phù hợp cho từng loại task trong workflow hàng ngày của bạn. Với task đơn giản như thế này, Haiku thường đủ tốt và tiết kiệm 75% chi phí.
