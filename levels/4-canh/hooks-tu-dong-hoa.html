Bài 4.3: Hooks & tự động hóa

⏱ ~7 phút đọc Level 4 Automation
💡
Nguyên lý

Hooks là lớp tự động hóa chạy ngầm — bạn không cần nhớ bước nào, Claude không cần nhắc bạn. Khi Claude edit file, hook chạy formatter tự động. Khi Claude xong task, hook gửi notification. Bạn định nghĩa một lần, hệ thống chạy mãi.

Hooks là gì và hoạt động như thế nào?

Hooks là shell commands được cấu hình để chạy tự động tại các điểm cụ thể trong workflow của Claude. Có 4 loại hook events:

  • PreToolUse — Chạy trước khi Claude dùng một tool. Dùng để block, validate, hoặc chuẩn bị.
  • PostToolUse — Chạy sau khi Claude dùng tool. Dùng để format, test, notify.
  • SessionStart — Chạy khi session Claude Code bắt đầu. Dùng để load context, check environment.
  • SessionStop — Chạy khi session kết thúc. Dùng để cleanup, save state, send summary.

Hook nhận thông tin về context qua environment variables: $FILE_PATH (file đang được edit), $TOOL_NAME (tool đang được dùng), $EXIT_CODE (kết quả của tool trước đó).

Cấu trúc cấu hình hooks

Hooks được định nghĩa trong ~/.claude/settings.json (global) hoặc .claude/settings.json (per-project):

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "npx prettier --write \"$FILE_PATH\" 2>/dev/null || true"
      },
      {
        "matcher": "Edit",
        "command": "bash ~/.claude/hooks/check-file-size.sh \"$FILE_PATH\""
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "bash ~/.claude/hooks/security-gate.sh \"$COMMAND\""
      }
    ],
    "SessionStop": [
      {
        "matcher": "*",
        "command": "bash ~/.claude/hooks/session-summary.sh"
      }
    ]
  }
}

Field matcher là tên tool hoặc pattern. Dùng * để match tất cả tools. Dùng Edit|Write để match nhiều tools.

5 production hooks phổ biến nhất

1
Auto-format sau khi edit PostToolUse

Chạy prettier/eslint sau mỗi lần Claude edit file. Code luôn đúng format mà không cần nhắc.

{
  "matcher": "Edit|Write",
  "command": "npx prettier --write \"$FILE_PATH\" 2>/dev/null || true"
}
2
Security gate — block sensitive files PreToolUse

Block Claude access vào .env, credentials, private keys. Hook return exit code 1 để ngăn tool execution.

# ~/.claude/hooks/security-gate.sh
FILE="$1"
BLOCKED_PATTERNS=(".env" "credentials" "id_rsa" "*.pem" "*.key")

for pattern in "${BLOCKED_PATTERNS[@]}"; do
  if [[ "$FILE" == *"$pattern"* ]]; then
    echo "BLOCKED: Access to $FILE denied by security policy"
    exit 1  # exit 1 = block tool execution
  fi
done
exit 0

Trong settings.json: "command": "bash ~/.claude/hooks/security-gate.sh \"$FILE_PATH\""

3
Auto-run tests sau khi code thay đổi PostToolUse

Chạy test suite sau mỗi lần Claude edit file code. Phát hiện regression ngay lập tức.

# Chỉ chạy test khi edit file TypeScript/JavaScript
{
  "matcher": "Edit",
  "command": "bash -c 'if [[ \"$FILE_PATH\" =~ \\.(ts|js)$ ]]; then npm test -- --passWithNoTests --silent 2>&1 | tail -5; fi'"
}
4
Git stash trước destructive operations PreToolUse

Tự động git stash trước khi Claude chạy commands có thể xóa data. Safety net tự động.

# ~/.claude/hooks/pre-bash-backup.sh
COMMAND="$1"
DANGEROUS_KEYWORDS=("rm -rf" "DROP TABLE" "DELETE FROM" "git reset --hard")

for keyword in "${DANGEROUS_KEYWORDS[@]}"; do
  if echo "$COMMAND" | grep -q "$keyword"; then
    echo "Auto-stashing before dangerous operation..."
    git stash push -m "auto-backup-$(date +%Y%m%d-%H%M%S)" 2>/dev/null || true
    break
  fi
done
exit 0
5
Slack notification khi task hoàn thành SessionStop

Gửi message Slack khi session Claude kết thúc — biết ngay khi agent xong việc dù bạn không nhìn màn hình.

# ~/.claude/hooks/session-summary.sh
WEBHOOK_URL="${SLACK_WEBHOOK_URL}"
if [ -z "$WEBHOOK_URL" ]; then exit 0; fi

MESSAGE="Claude Code session completed at $(date '+%H:%M %d/%m/%Y')"
curl -s -X POST "$WEBHOOK_URL" \
  -H 'Content-type: application/json' \
  --data "{\"text\":\"$MESSAGE\"}" \
  > /dev/null 2>&1

Hooks vs Skills: Khi nào dùng cái nào?

Dùng Hooks khi: Bạn muốn hành động xảy ra tự động, luôn luôn, không cần trigger thủ công. Format code, chạy tests, block sensitive files — những thứ này là "luật" của workflow, không phải "tùy chọn".

Dùng Skills khi: Bạn muốn Claude có khả năng mới, gọi khi cần. Tạo báo cáo, xử lý data, generate content — những thứ này là "công cụ" trong toolbox, dùng khi có nhu cầu cụ thể.

Kết hợp tốt nhất: Skill định nghĩa workflow, Hook enforce quality gates trong workflow đó. Ví dụ: skill weekly-report tạo báo cáo, hook PostToolUse đảm bảo file output luôn được format đúng.

Debugging hooks

Khi hook không hoạt động như mong đợi:

1. Kiểm tra exit code: Hook return 0 = success (tiếp tục), return 1 = block tool. Nếu hook của bạn vô tình return 1, Claude sẽ bị block. Luôn có || true hoặc || exit 0 cho các commands không quan trọng.

2. Test script độc lập: Trước khi đưa vào settings.json, chạy script trực tiếp từ terminal với test inputs để confirm nó hoạt động.

3. Add logging: Thêm echo "[HOOK] triggered: $FILE_PATH" >> /tmp/claude-hooks.log vào đầu script để trace khi nào hook được gọi.

Pitfall phổ biến: Environment variables trong hook script không giống environment của shell thông thường. Nếu cần PATH hoặc env vars cụ thể, source chúng explicitly: source ~/.bashrc hoặc dùng absolute paths cho mọi executables.

Thực hành

Tạo hook đơn giản: sau mỗi lần Claude edit file, tự động đếm số dòng và warn nếu vượt 200 dòng. Tạo file ~/.claude/hooks/check-file-size.sh với nội dung: đọc $FILE_PATH, đếm dòng bằng wc -l, nếu > 200 in cảnh báo ra stdout. Thêm vào ~/.claude/settings.json như PostToolUse hook. Thử edit một file lớn và xem output.
