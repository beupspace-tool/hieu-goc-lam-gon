Bài 4.1: Xây dựng Skill tùy chỉnh

⏱ ~8 phút đọc Level 4 Power user
💡
Nguyên lý

Skill là cách bạn dạy Claude domain knowledge cụ thể của doanh nghiệp mình — một lần, dùng mãi. Thay vì giải thích lại quy trình mỗi lần chat, bạn đóng gói nó thành skill. Claude đọc skill → hiểu ngữ cảnh → làm đúng ngay từ đầu.

Khi nào cần custom skill?

Không phải mọi workflow đều cần skill. Nhưng có ba dấu hiệu rõ ràng cho thấy đã đến lúc:

1. Bạn lặp lại cùng workflow hơn 3 lần. Nếu bạn đang copy-paste prompt dài vào Claude mỗi tuần để làm báo cáo doanh thu, đó là skill candidate. Thời gian bạn tốn copy-paste là thời gian lãng phí có thể loại bỏ hoàn toàn.

2. Team cần chuẩn hóa cách làm. Khi 3 người trong team đang dùng Claude theo 3 cách khác nhau cho cùng 1 task, kết quả sẽ không nhất quán. Skill là cách enforce standard — mọi người gọi cùng 1 skill, ra cùng 1 format output.

3. Bạn muốn Claude "biết" domain knowledge cụ thể. Claude mặc định không biết cấu trúc database của bạn, không biết tone of voice thương hiệu, không biết quy trình onboarding khách hàng của công ty. Skill là nơi bạn nạp những kiến thức này vào.

Anatomy của SKILL.md

Mỗi skill là một thư mục chứa file SKILL.md — đây là "bộ não" của skill. Cấu trúc chuẩn:

---
name: invoice-processor
description: Process and analyze invoices from CSV/PDF files
allowed-tools: [Bash, Read, Write, Edit, Glob, Grep]
---

# Invoice Processor

## Purpose
Đọc file hóa đơn (CSV hoặc PDF), trích xuất thông tin, tạo báo cáo summary.

## Input
- File CSV hoặc PDF chứa danh sách hóa đơn
- Ngày bắt đầu và kết thúc cần báo cáo

## Output
- File `invoice-report-{date}.md` chứa:
  - Tổng số hóa đơn
  - Tổng giá trị
  - Breakdown theo khách hàng
  - Hóa đơn quá hạn (nếu có)

## Process
1. Đọc file input với Read tool
2. Parse dữ liệu (dùng script nếu cần)
3. Tính toán aggregates
4. Tạo báo cáo markdown

## Notes
- Định dạng tiền tệ: VND, dấu phẩy phân cách hàng nghìn
- Nếu field trống, ghi "N/A"
- Hóa đơn quá hạn = ngày đến hạn < hôm nay

Phần YAML frontmatter ở đầu file quan trọng — name là cách bạn invoke, description giúp Claude detect khi nào nên dùng skill này, allowed-tools giới hạn những gì skill được phép làm.

Cấu trúc thư mục skill

Một skill đầy đủ có 4 thành phần:

~/.claude/skills/invoice-processor/
├── SKILL.md ← bộ não
├── scripts/
│ ├── parse-csv.py ← xử lý dữ liệu
│ └── pdf-extract.py
├── references/
│ ├── invoice-schema.md ← cấu trúc dữ liệu
│ └── sample-output.md ← ví dụ output mong muốn
└── data/
└── customer-list.csv ← dữ liệu tham chiếu

scripts/ — Python hoặc bash scripts mà skill gọi khi cần xử lý phức tạp. Claude sẽ chạy python3 scripts/parse-csv.py input.csv thay vì tự parse. Tách logic phức tạp ra scripts giúp skill dễ maintain và test.

references/ — Docs, examples, templates mà Claude đọc để hiểu context. Đây là nơi bạn đặt domain knowledge: cấu trúc dữ liệu của bạn, format output mong muốn, business rules, style guide. Claude đọc references khi cần thêm thông tin để hoàn thành task.

data/ — Dữ liệu tham chiếu tĩnh: danh sách khách hàng, bảng giá, config. Không thay đổi thường xuyên nhưng skill cần access.

💡
Nguyên lý thiết kế

SKILL.md chỉ nên dài 50–150 dòng. Nếu dài hơn, tách logic ra references/. SKILL.md là "hợp đồng" — mô tả purpose, input, output, process. Chi tiết implementation đặt ở scripts và references. Claude sẽ biết cần đọc gì khi nào.

Step-by-step: Tạo skill "weekly-report"

Chúng ta sẽ build skill tự động tạo weekly report từ data files. Đây là skill thực tế, không phải demo.

Bước 1: Tạo thư mục

mkdir -p ~/.claude/skills/weekly-report/scripts
mkdir -p ~/.claude/skills/weekly-report/references

Bước 2: Viết SKILL.md

---
name: weekly-report
description: Generate weekly business report from data files
allowed-tools: [Bash, Read, Write, Glob, Grep]
---

# Weekly Report Generator

## Purpose
Tạo báo cáo tuần từ các file dữ liệu trong thư mục dự án.

## Trigger
- Khi user nói "tạo báo cáo tuần" hoặc gọi /weekly-report
- Tự động detect tuần hiện tại

## Input sources
- git log: commits tuần này
- data/metrics.csv: số liệu KPI (nếu tồn tại)
- notes/weekly-notes.md: ghi chú thủ công (nếu tồn tại)

## Output
File `reports/weekly-{YYYY-WW}.md` gồm:
1. Summary (3 bullet points)
2. Completed tasks (từ git log)
3. Key metrics (từ metrics.csv)
4. Next week priorities

## Template
Xem references/report-template.md

Bước 3: Tạo template trong references/

# Weekly Report — Tuần {week}, {year}

## Tóm tắt
- [3 điểm nổi bật nhất trong tuần]

## Đã hoàn thành
{danh sách từ git log, nhóm theo feature}

## Số liệu
{metrics từ CSV, format: Metric | Tuần này | Tuần trước | Thay đổi}

## Tuần tới
- [ ] {priority 1}
- [ ] {priority 2}
- [ ] {priority 3}

Bước 4: Invoke skill

Sau khi tạo xong, có hai cách gọi skill:

  • Gõ /weekly-report trong Claude Code — Claude tự load skill và thực thi
  • Mô tả task tự nhiên: "Tạo báo cáo tuần cho tôi" — Claude detect skill phù hợp qua description

Scripts: Khi nào cần và viết thế nào

Không phải skill nào cũng cần scripts. Dùng scripts khi:

  • Cần parse file format phức tạp (PDF, Excel, JSON lồng nhau)
  • Logic tính toán phức tạp hơn Claude có thể tự làm trong context
  • Cần gọi API external
  • Performance — Python nhanh hơn Claude tự parse qua text

Ví dụ script đơn giản để extract git log:

#!/usr/bin/env python3
# scripts/get-git-log.py
# Lấy commits từ 7 ngày qua, format cho report

import subprocess
import sys
from datetime import datetime, timedelta

since = (datetime.now() - timedelta(days=7)).strftime('%Y-%m-%d')
result = subprocess.run(
    ['git', 'log', f'--since={since}', '--oneline', '--no-merges'],
    capture_output=True, text=True
)
print(result.stdout)

Claude gọi script này bằng Bash tool: python3 ~/.claude/skills/weekly-report/scripts/get-git-log.py

Testing skill của bạn

Trước khi dùng cho production, test skill theo 3 bước:

1. Kiểm tra SKILL.md hợp lệ — Đảm bảo YAML frontmatter đúng format, không có syntax error. Mở Claude Code, gõ /weekly-report — nếu skill load được, bạn sẽ thấy Claude acknowledge nó.

2. Test với edge cases — Chạy skill khi không có file metrics.csv, khi git repo chưa có commit tuần này. Skill tốt xử lý gracefully, không crash.

3. Kiểm tra output format — So sánh output với template trong references/. Nếu format sai, cập nhật SKILL.md với ví dụ rõ hơn hoặc thêm format rules.

Community skills đáng tham khảo

Thay vì build từ đầu, hãy xem qua các skills đã được community tạo sẵn:

  • awesome-claude-code — 135+ agents và skills, nhiều use cases thực tế
  • Skills cho code review, documentation, testing, deployment trên GitHub
  • Tìm skill gần nhất với nhu cầu → copy → customize cho context của bạn

Nguyên tắc DRY áp dụng cho cả skills: đừng viết lại thứ người khác đã viết tốt rồi.

⚠️
Lưu ý bảo mật

Skills có thể chứa allowed-tools: [Bash] — nghĩa là Claude có thể chạy shell commands khi skill được invoke. Đừng đặt credentials hoặc API keys trực tiếp trong SKILL.md — dùng environment variables hoặc tham chiếu đến file .env riêng. Audit skills từ community trước khi chạy, đặc biệt là phần scripts/.

🔧
Thực hành

Tạo skill đơn giản: daily-standup — Claude đọc git log hôm nay, tóm tắt changes, tạo standup report 3 phần (Hôm qua làm gì, Hôm nay làm gì, Blockers). Đặt vào ~/.claude/skills/daily-standup/SKILL.md. Mở Claude Code trong một project có git history, gõ /daily-standup. Cả skill chỉ cần khoảng 30 dòng SKILL.md — không cần scripts cho bước đầu.
