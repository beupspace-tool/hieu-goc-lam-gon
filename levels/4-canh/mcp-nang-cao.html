Bài 4.2: MCP nâng cao

⏱ ~7 phút đọc Level 4 Power user
💡
Nguyên lý

MCP (Model Context Protocol) là cầu nối giữa Claude và hệ thống ngoài. Ở Level 2 bạn dùng MCP servers sẵn có. Ở Level 4, bạn build MCP server riêng để Claude trực tiếp access internal API, database, hay bất kỳ hệ thống nào của doanh nghiệp — mà không cần copy-paste data thủ công.

Multi-server setup: Chạy nhiều MCP cùng lúc

Claude Code có thể kết nối với nhiều MCP servers đồng thời. Mỗi server phục vụ một mục đích khác nhau, Claude tự chọn server phù hợp dựa trên task.

File .mcp.json trong project root định nghĩa tất cả servers:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/projects"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "notion": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-notion"],
      "env": {
        "NOTION_API_KEY": "${NOTION_API_KEY}"
      }
    },
    "internal-crm": {
      "command": "python3",
      "args": ["/path/to/crm-mcp-server/server.py"],
      "env": {
        "CRM_API_URL": "https://api.yourcrm.internal",
        "CRM_API_KEY": "${CRM_API_KEY}"
      }
    }
  }
}

Khi bạn hỏi Claude "lấy thông tin khách hàng Nguyễn Văn A", Claude nhận ra cần dùng internal-crm server. Khi hỏi "tạo PR trên GitHub", Claude dùng github server. Routing tự động — bạn không cần chỉ định.

MCP resources vs tools: Phân biệt quan trọng

Tools — Thực hiện actions

  • Create, update, delete data
  • Send messages, emails
  • Trigger workflows
  • Query với side effects
  • Ví dụ: create_ticket(), update_status()

Resources — Đọc data

  • Read-only access
  • Structured data sources
  • No side effects
  • Cacheable
  • Ví dụ: customer_list, product_catalog

Nguyên tắc thiết kế: expose Resources cho dữ liệu đọc thường xuyên (Claude cache được, tiết kiệm token), Tools cho actions cần xác nhận hoặc có side effects.

Build custom MCP server với Python (FastMCP)

FastMCP là framework Python đơn giản nhất để build MCP server. Ví dụ dưới đây tạo server cho internal CRM:

# crm-mcp-server/server.py
from fastmcp import FastMCP
import requests
import os

mcp = FastMCP("Internal CRM")

CRM_URL = os.environ["CRM_API_URL"]
CRM_KEY = os.environ["CRM_API_KEY"]
HEADERS = {"Authorization": f"Bearer {CRM_KEY}"}

@mcp.tool()
def get_customer(customer_id: str) -> dict:
    """Lấy thông tin chi tiết của một khách hàng theo ID."""
    response = requests.get(
        f"{CRM_URL}/customers/{customer_id}",
        headers=HEADERS,
        timeout=10
    )
    response.raise_for_status()
    return response.json()

@mcp.tool()
def search_customers(query: str, limit: int = 10) -> list:
    """Tìm kiếm khách hàng theo tên, email hoặc số điện thoại."""
    response = requests.get(
        f"{CRM_URL}/customers/search",
        params={"q": query, "limit": limit},
        headers=HEADERS,
        timeout=10
    )
    response.raise_for_status()
    return response.json()["results"]

@mcp.tool()
def update_customer_status(customer_id: str, status: str) -> dict:
    """Cập nhật trạng thái khách hàng (active/inactive/vip)."""
    response = requests.patch(
        f"{CRM_URL}/customers/{customer_id}",
        json={"status": status},
        headers=HEADERS,
        timeout=10
    )
    response.raise_for_status()
    return {"success": True, "customer_id": customer_id, "new_status": status}

@mcp.resource("crm://customers/list")
def customer_list() -> str:
    """Danh sách tất cả khách hàng active — read-only resource."""
    response = requests.get(
        f"{CRM_URL}/customers?status=active&limit=500",
        headers=HEADERS,
        timeout=15
    )
    return response.text

if __name__ == "__main__":
    mcp.run()

Cài FastMCP và chạy:

pip install fastmcp requests
python3 server.py

Sau đó thêm vào .mcp.json như ví dụ ở trên. Restart Claude Code. Bây giờ bạn có thể nói "tìm khách hàng tên Minh" và Claude sẽ gọi search_customers("Minh").

TypeScript MCP server (MCP SDK)

Nếu team bạn dùng Node.js, TypeScript SDK sẽ phù hợp hơn:

// inventory-mcp/index.ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({ name: "Inventory System", version: "1.0.0" });

server.tool(
  "check_stock",
  "Kiểm tra tồn kho của sản phẩm theo SKU",
  { sku: z.string().describe("Mã sản phẩm SKU") },
  async ({ sku }) => {
    const stock = await fetchStockFromDB(sku);
    return {
      content: [{
        type: "text",
        text: JSON.stringify({ sku, quantity: stock.quantity, warehouse: stock.location })
      }]
    };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);

Authentication patterns

API Key (đơn giản nhất): Truyền qua environment variable trong .mcp.json. Dùng ${ENV_VAR_NAME} syntax — Claude Code sẽ resolve từ shell environment của bạn. Không bao giờ hardcode key trong file JSON.

OAuth2: Cho APIs cần token refresh (Google, Microsoft). Build token refresh logic trong server, lưu refresh token vào file local (không trong source code):

# Trong MCP server Python
def get_access_token():
    """Tự động refresh nếu token hết hạn."""
    token_data = load_token_from_file("~/.crm-tokens.json")
    if is_expired(token_data):
        token_data = refresh_oauth_token(token_data["refresh_token"])
        save_token_to_file(token_data, "~/.crm-tokens.json")
    return token_data["access_token"]

Performance và production considerations

MCP servers chạy local trên máy bạn — latency rất thấp so với APIs qua internet. Nhưng khi đưa vào production:

  • Health checks: Add /health endpoint. Nếu server crash, Claude nhận được lỗi rõ ràng thay vì timeout.
  • Retry logic: Network errors xảy ra. Implement exponential backoff cho tất cả HTTP calls.
  • Timeout: Set timeout cho mọi request. Default là vô hạn — một query chậm sẽ block Claude.
  • Logging: Log mọi tool call với timestamp. Khi debug, bạn cần biết Claude đã gọi gì và nhận được gì.

Security best practices

Principle of least privilege: MCP server cho marketing team không cần access vào finance data. Tạo separate servers với separate API keys có permissions phù hợp — đừng dùng một master key cho tất cả.

Audit logs: Log mọi mutation (create, update, delete) với: timestamp, tool được gọi, parameters, user context (nếu có). Khi xảy ra sự cố, bạn cần trace được Claude đã làm gì.

Separate environments: Server dev dùng staging database, server production dùng production database. Đừng test với production data.

⚠️
Lưu ý quan trọng

MCP servers với destructive tools (delete, update) cần confirmation flow. Thêm parameter confirm: bool = False vào tools nguy hiểm — nếu confirm=False, trả về preview thay vì execute. Claude sẽ hỏi bạn xác nhận trước khi thực sự xóa data.

🔧
Thực hành

Tạo file .mcp.json trong một project folder với 3 servers: filesystem (trỏ vào thư mục project), github (cần GitHub token), và một server khác bạn tự chọn từ MCP servers registry. Restart Claude Code và thử query mỗi server bằng câu hỏi tự nhiên — quan sát Claude chọn đúng server cho mỗi loại query.
