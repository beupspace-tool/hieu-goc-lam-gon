Bài 0.4: Terminal — Công cụ giao tiếp bằng chữ

⏱ ~5 phút đọc Level 0
💡
Nguyên lý

GUI (giao diện đồ hoạ) giới hạn bạn vào những gì nhà thiết kế đã lường trước. CLI (command line) không có giới hạn đó — bạn có thể làm bất cứ điều gì hệ thống cho phép. Claude Code có thể dùng qua Desktop App (giao diện đồ hoạ, dễ nhất), VS Code, hoặc Terminal. Bài này giúp bạn hiểu terminal — không bắt buộc phải dùng, nhưng biết sẽ giúp bạn tự tin hơn khi cần.

GUI vs Terminal — không phải chọn một, bỏ một

Claude Code Desktop App đã là GUI — bạn hoàn toàn có thể dùng mà không cần biết terminal. Nhưng hiểu terminal giúp bạn hiểu Claude đang làm gì "bên dưới" khi nó chạy lệnh thay bạn.

Giao diện đồ hoạ tốt vì giúp bắt đầu nhanh — bạn thấy nút, bạn nhấn, bạn có kết quả. Nhưng mọi hành động trong GUI đều phải được lập trình viên dự đoán trước. Terminal không có giới hạn đó — khi cần xử lý 10.000 dòng dữ liệu, đổi tên 500 file theo pattern, hoặc kết nối với API bên ngoài, terminal làm được. Và Claude Code sẽ chạy các lệnh terminal này thay bạn — bạn chỉ cần hiểu kết quả.

🖱️ GUI — Giao diện đồ hoạ

  • Tốt cho: tác vụ thường gặp, người dùng mới
  • Giới hạn bởi những gì nhà thiết kế tạo ra
  • Khó tự động hoá và script hoá
  • Không thể kết nối với công cụ khác dễ dàng

⌨️ CLI — Dòng lệnh

  • Tốt cho: tự động hoá, tác vụ phức tạp, tích hợp hệ thống
  • Không giới hạn — làm được bất cứ gì hệ thống cho phép
  • Scriptable: viết một lần, chạy ngàn lần
  • Kết nối dễ với Git, databases, APIs, cloud services

Terminal trong thực tế

Đây là terminal điển hình — môi trường Claude Code chạy trong đó:

Terminal — bash
user@macbook ~ $ pwd
/Users/user
user@macbook ~ $ ls Documents/
bao-cao-q1-2026.xlsx hop-dong/ marketing/
user@macbook ~ $ mkdir projects/landing-page
user@macbook ~ $ cat brief-cong-ty.txt
Tên công ty: ABC Consulting. Dịch vụ: tư vấn...
user@macbook ~ $

Năm lệnh đủ để làm việc với Claude Code

Bạn không cần học terminal từ đầu. Năm lệnh sau đủ để điều hướng và làm việc cơ bản:

  • pwd — "print working directory" — bạn đang ở thư mục nào. Dùng khi bị lạc.
  • ls — liệt kê file và thư mục trong vị trí hiện tại. ls -la hiển thị thêm chi tiết bao gồm file ẩn.
  • cd tên-thư-mục — di chuyển vào thư mục. cd .. lùi lại một cấp.
  • mkdir tên-thư-mục — tạo thư mục mới.
  • cat tên-file — xem nội dung file văn bản ngay trong terminal.

Environment variables và paths — biết để không bị bối rối

Path là địa chỉ đầy đủ của file hoặc thư mục. /Users/user/Documents/bao-cao.xlsx là path tuyệt đối. ./Documents/bao-cao.xlsx là path tương đối từ vị trí hiện tại. Claude Code thường làm việc với path tương đối trong project của bạn.

Environment variables là biến hệ thống lưu thông tin cấu hình — API keys, đường dẫn quan trọng, settings. Ví dụ: ANTHROPIC_API_KEY=sk-ant-... là environment variable chứa key để Claude Code xác thực với Anthropic. Bạn không nhập key trực tiếp vào code — bạn lưu vào environment variable, an toàn hơn và dễ quản lý hơn.

🔧
Thực hành — 3 phút

Mở Terminal (Mac: Cmd+Space → "Terminal" | Windows: tìm "Windows Terminal" hoặc "PowerShell"). Gõ từng lệnh và quan sát kết quả: (1) pwd — xem bạn đang ở đâu; (2) ls — xem có gì trong thư mục; (3) cd Desktop rồi ls lại — xem file trên màn hình Desktop của bạn. Bạn vừa điều hướng hệ thống file bằng command line. Claude Code sẽ làm điều này hàng trăm lần trong một phiên làm việc điển hình.

⚠️
Lệnh không thể hoàn tác

Một số lệnh terminal không có "undo". rm -rf xóa file vĩnh viễn, không qua Trash. Claude Code biết điều này và sẽ luôn xin phép trước khi thực thi lệnh có tác động không thể đảo ngược. Nguyên tắc quan trọng: đọc kỹ trước khi nhấn Enter, đặc biệt với lệnh xóa hoặc ghi đè file.
