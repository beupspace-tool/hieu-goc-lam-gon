Bài 0.5: "Code" là gì?

⏱ ~4 phút đọc Level 0
💡
Nguyên lý

Logic kinh doanh bạn đang áp dụng mỗi ngày — "nếu đơn hàng trên 5 triệu thì miễn phí vận chuyển", "nếu khách mua 3 lần thì upgrade lên VIP" — chính là code. Bạn đã suy nghĩ theo cấu trúc lập trình từ lâu, chỉ chưa biết gọi nó bằng tên đó. Hiểu điều này giúp bạn mô tả yêu cầu cho Claude Code chính xác hơn nhiều.

Code là logic kinh doanh được formalised

Khi bạn viết quy trình vận hành cho nhân viên: "Nếu đơn hàng trên 2 triệu đồng, áp mã giảm giá 10%. Nếu khách VIP, áp thêm 5% nữa. Nếu cả hai, cộng lại thành 15%." — bạn vừa viết logic điều kiện. Trong Python, nó trông như thế này:

🐍 Python — logic giảm giá đơn hàng
def tinh_giam_gia(tong_don_hang, la_khach_vip): giam_gia = 0 if tong_don_hang > 2000000: giam_gia += 0.10 if la_khach_vip: giam_gia += 0.05 return tong_don_hang * (1 - giam_gia) # Đơn hàng 3 triệu, khách VIP → giảm 15% print(tinh_giam_gia(3000000, True)) # Kết quả: 2550000

Nếu bạn đọc được logic đó — và bạn đọc được vì nó giống hệt cách bạn nghĩ về kinh doanh — thì khoảng cách giữa bạn và code không lớn như bạn nghĩ.

SQL — ngôn ngữ bạn nên biết đọc ngay hôm nay

SQL (Structured Query Language) là cách nói chuyện với database. Mọi dữ liệu kinh doanh của bạn — đơn hàng, khách hàng, kho hàng — đều nằm trong database và SQL là cách truy xuất chúng.

⚙️ SQL — câu hỏi kinh doanh thực tế
-- Đơn hàng nào trên 5 triệu trong tháng 3? SELECT ma_don_hang, ten_khach, tong_tien FROM don_hang WHERE tong_tien > 5000000 AND thang = 3 ORDER BY tong_tien DESC; -- Top 10 sản phẩm bán chạy nhất Q1? SELECT ten_sp, SUM(so_luong) AS tong_ban FROM chi_tiet_don_hang GROUP BY ten_sp ORDER BY tong_ban DESC LIMIT 10;

SQL đọc gần như tiếng Anh tự nhiên. Bạn không cần viết SQL — Claude Code viết thay bạn. Nhưng khi nhìn vào kết quả báo cáo bất thường, biết đặt câu hỏi đúng giúp Claude Code tạo query chính xác hơn.

Kiến trúc ba lớp của mọi ứng dụng web

Mọi website hoặc app kinh doanh đều có ba lớp. Hiểu điều này giúp bạn phân bổ ngân sách và đặt câu hỏi đúng khi thuê agency:

🏗️
Frontend (HTML/CSS/JS)
Giao diện người dùng thấy. Màu sắc, layout, tương tác. Chi phí: ~20–40% dự án.
⚙️
Backend (Python/Node/Java)
Logic kinh doanh, xử lý đơn hàng, tính toán. Chi phí: ~40–60% dự án.
🗄️
Database (SQL/NoSQL)
Lưu trữ tất cả dữ liệu. Thiết kế database tốt quyết định hiệu suất hệ thống.

Khi agency báo giá "làm website", hỏi rõ: lớp nào họ đang làm, lớp nào là template có sẵn, và bạn có quyền sở hữu source code không sau khi dự án kết thúc.

🔧
Thực hành — đọc code thực

Đọc đoạn SQL sau và trả lời: nó đang hỏi gì?
SELECT ten_khach, COUNT(*) as so_don FROM don_hang WHERE trang_thai = 'completed' GROUP BY ten_khach HAVING COUNT(*) >= 5 ORDER BY so_don DESC;

Nếu bạn đọc được: "Tìm những khách hàng đã hoàn thành ít nhất 5 đơn, sắp xếp theo số đơn nhiều nhất" — bạn đã đọc được SQL. Đây là câu truy vấn thực tế để tìm khách hàng trung thành cho chương trình loyalty.

⚠️
Source code là tài sản doanh nghiệp

Khi thuê agency xây dựng phần mềm hoặc website, luôn yêu cầu bàn giao source code (code gốc) trong hợp đồng. Nhiều doanh nghiệp Việt bị "con tin" bởi agency vì không có source code — mỗi lần cần sửa nhỏ phải trả tiền cho agency cũ, không thể chuyển sang đội khác. Source code là tài sản kỹ thuật số của bạn, không khác gì bản vẽ nhà hay sổ đỏ.
