Bài 5.1: Thiết kế hệ thống cho team

⏱ ~8 phút đọc Level 5 System Design
🏗️
Nguyên lý

Hệ thống tốt là hệ thống mà mọi thành viên có thể contribute ngay lập tức. CLAUDE.md + Skills + MCP = onboarding trong 1 ngày thay vì 1 tháng. Khi bạn thiết kế hệ thống đúng, năng lực của mỗi người nhân lên — không chỉ cộng lại.

CLAUDE.md = Team DNA

Ở Level 1, bạn học CLAUDE.md như một file hướng dẫn cá nhân. Ở Level 5, bạn hiểu nó là thứ gì đó sâu hơn: CLAUDE.md là nơi encode văn hóa làm việc của team.

Mỗi khi Claude Code đọc CLAUDE.md, nó không chỉ nhận hướng dẫn kỹ thuật — nó hấp thụ toàn bộ ngữ cảnh về cách team bạn vận hành: coding style, PR process, deployment checklist, naming conventions, business context, stakeholder preferences. Khi team member mới join, họ không cần 3 tuần shadow người khác — họ chỉ cần đọc CLAUDE.md và bắt đầu làm việc có ý nghĩa từ ngày đầu tiên.

Đây là sự khác biệt giữa tổ chức dùng Claude Code như công cụ cá nhân và tổ chức dùng nó như hạ tầng chiến lược.

3-Tier Architecture cho organization

Một tổ chức vận hành tốt với Claude Code cần ba lớp CLAUDE.md lồng vào nhau, mỗi lớp xử lý một phạm vi khác nhau:

GLOBAL
~/.claude/CLAUDE.md — Company-wide standards

Ngôn ngữ output mặc định (tiếng Việt hay English?), security policy (không commit secrets, không push to main trực tiếp), commit message format (conventional commits), tone of voice khi viết content, compliance requirements. Áp dụng cho mọi project, mọi người trong tổ chức.

PROJECT
./CLAUDE.md — Project-specific context

Tech stack đang dùng (Next.js 14, Tailwind, Supabase), kiến trúc tổng thể, business context (đây là gì, phục vụ ai, KPIs quan trọng), deployment process, environment variables cần có, đường dẫn đến tài liệu quan trọng. Bất kỳ developer nào clone repo này và đọc CLAUDE.md có thể hiểu project trong 10 phút.

MODULE
./src/CLAUDE.md — Module-specific patterns

API patterns riêng của module (authentication flow, error handling), data models và schema, đặc thù business logic cần biết, ví dụ code cho các patterns phổ biến. Dành cho codebase lớn nơi mỗi module có complexity riêng.

Khi Claude Code làm việc trong một file cụ thể, nó tự động đọc và merge context từ tất cả ba lớp. Không cần nhắc lại, không cần giải thích. Mọi thứ đã được encode sẵn.

Skills library cho team

Skills là các workflow có thể tái sử dụng — bạn đã học điều này ở Level 3. Nhưng ở cấp độ tổ chức, skills library trở thành tài sản chiến lược.

Nguyên tắc: mỗi khi bạn lặp lại một workflow hơn 3 lần, hãy biến nó thành skill. Đây là ví dụ về internal skills library cho một team marketing:

  • /content-review — Review draft theo brand voice, check SEO, suggest improvements
  • /social-schedule — Tạo calendar post từ article, format cho từng platform
  • /report-weekly — Pull metrics từ Notion/Sheets, tạo weekly report template
  • /competitor-watch — Phân tích competitor content, tìm gap và opportunity
  • /deploy-check — Pre-deployment checklist: tests, security scan, environment vars

Khi skill được chia sẻ trong team, toàn bộ team được hưởng lợi từ best practices của một người. Người giỏi nhất không phải làm hết — họ encode kiến thức vào hệ thống.

MCP infrastructure cho team

MCP servers kết nối Claude Code với công cụ bên ngoài. Ở cấp độ team, bạn cần một template chuẩn để mọi người cùng dùng cùng một set of tools:

File .mcp.json template cho team (commit vào repo, không chứa secrets):

{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "NOTION_API_KEY": "${NOTION_API_KEY}"
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "slack": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "${SLACK_BOT_TOKEN}"
      }
    }
  }
}

Secrets được inject qua environment variables — không bao giờ hardcode trong file. Mỗi team member tạo file .env.local riêng với API keys của mình (và file này trong .gitignore). Template không chứa secrets nhưng định nghĩa đầy đủ infrastructure.

Permission model

Claude Code có permission system để kiểm soát những gì agent được phép làm. Ở cấp độ team, điều này rất quan trọng:

Developer permissions (file .claude/settings.json trong repo):

  • Cho phép: đọc/viết files trong project, chạy tests, build commands
  • Không cho phép: push to main, delete production data, deploy trực tiếp

Admin permissions (override tại user level):

  • Deployment commands
  • Database migrations
  • Infrastructure changes

Với Claude Code Team/Enterprise plan, bạn có RBAC (Role-Based Access Control) và audit logging đầy đủ — mọi action đều được ghi lại, ai làm gì, lúc nào.

Onboarding workflow

Đây là kết quả khi hệ thống được thiết kế đúng. New member join team:

  1. Ngày 1, giờ 1: Clone repo. CLAUDE.md tự giải thích project context, tech stack, business goals.
  2. Ngày 1, giờ 2: Setup MCP servers theo template — mọi tools kết nối trong 30 phút.
  3. Ngày 1, giờ 3: Chạy skills library — đã có sẵn /content-review, /deploy-check, workflows chuẩn.
  4. Ngày 1, buổi chiều: Làm task thực tế đầu tiên với Claude Code. Có output.

So với onboarding truyền thống: 2-4 tuần shadow, đọc wiki lỗi thời, hỏi đồng nghiệp liên tục. CLAUDE.md không bao giờ out-of-date vì developer cập nhật nó khi code thay đổi.

Đo lường hiệu quả

Hệ thống tốt cần metrics để validate. Sau khi deploy Claude Code cho team, track:

↓40%
Time-to-merge trung bình
↓60%
Thời gian onboarding
↑3x
Content velocity
↓25%
Bug rate

Những con số này từ các team thực tế đã triển khai Claude Code với hệ thống đầy đủ. Kết quả sẽ khác nhau tùy context, nhưng trend là nhất quán: hệ thống tốt nhân rộng năng lực của cả team.

⚠️
Lỗi phổ biến

Nhiều team setup CLAUDE.md một lần rồi không cập nhật. CLAUDE.md phải là living document — cập nhật khi code thay đổi, khi business context thay đổi, khi team học được bài học mới. Assign ownership: ai là người chịu trách nhiệm maintain CLAUDE.md? Đó phải là công việc thực sự, không phải việc phụ.

🔧
Thực hành

Thiết kế hệ thống Claude Code cho team bạn: (1) Viết CLAUDE.md với 3 tiers — global standards, project context, module specifics. (2) Liệt kê 5 skills team cần ngay bây giờ. (3) Xác định MCP servers cần setup — Notion? GitHub? Slack? Internal tools? (4) Viết onboarding checklist Day 1 cho member mới. Sau đó test với một người thực tế và ghi lại điểm friction.
