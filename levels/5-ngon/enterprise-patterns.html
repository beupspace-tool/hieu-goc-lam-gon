Bài 5.3: Enterprise patterns

⏱ ~8 phút đọc Level 5 Enterprise
🏢
Nguyên lý

Scale thay đổi mọi thứ. Những gì hoạt động tốt cho 1 người đôi khi cần kiến trúc khác cho 50 người. Bài này về cách thiết kế Claude Code deployment để hoạt động ở quy mô team lớn, với đầy đủ security, cost management, và auditability.

Parallel development at scale

Vấn đề phổ biến ở team developer lớn: nhiều người cùng làm nhiều feature, nhưng Claude Code sessions không thể share context với nhau. Giải pháp là git worktrees.

Git worktrees cho phép bạn có nhiều working directory từ cùng một repository — mỗi directory ở một branch khác nhau. Mỗi developer (hoặc Claude session) làm việc trong worktree riêng, không conflict với nhau:

# Setup worktrees cho parallel development
git worktree add ../feature-auth feature/authentication
git worktree add ../feature-dashboard feature/dashboard
git worktree add ../hotfix-payment hotfix/payment-bug

# Mỗi folder là một Claude Code session riêng biệt
# cd ../feature-auth && claude  → Session 1
# cd ../feature-dashboard && claude  → Session 2
# cd ../hotfix-payment && claude  → Session 3

5–20 concurrent sessions là khả thi với pattern này. CI/CD validates mỗi branch độc lập. Merge to main khi tests pass. Không có conflict vì mỗi session chỉ touch files của feature nó đang làm.

GitHub Actions integration

Claude Code tích hợp với GitHub Actions để tạo automation pipeline. Workflow phổ biến nhất:

💬 @claude mention trên Issue
→
🔍 Claude investigates codebase
→
🔧 Tạo PR với fix
→
✅ CI runs tests
→
👀 Human review & merge

Cụ thể hơn: khi developer mention @claude trong GitHub Issue với mô tả bug, Claude Code agent tự động:

  1. Đọc issue và reproduce vấn đề
  2. Tìm root cause trong codebase
  3. Implement fix
  4. Tạo PR với description đầy đủ và tests
  5. CI chạy — nếu pass, human review và merge

Setup qua claude.yml GitHub Actions workflow. Yêu cầu GitHub Actions permission và Anthropic API key trong repository secrets.

Ngoài bug fixing, có thể setup: automated code review khi có PR, documentation generation từ code changes, test suite expansion khi thêm new features.

Security & compliance

🔐 SSO Integration

Team/Enterprise plan hỗ trợ SAML SSO — nhân viên login bằng company credentials, không cần quản lý account riêng. Offboarding: deactivate trong IdP là đủ.

👥 RBAC

Ba role cơ bản: Admin (full access, billing, settings), Developer (run Claude, manage own projects), Viewer (read outputs, no execution). Custom roles với Enterprise.

📋 Audit Logging

Track tất cả Claude actions: ai làm gì, lúc nào, trên project nào. Export log để compliance reporting. Essential cho regulated industries (finance, healthcare).

🏠 Data Residency

Enterprise plan có data residency options (US, EU). Anthropic policy: không train model trên data của bạn. Prompts và outputs không dùng để improve models.

💡
Data Privacy

Anthropic không dùng conversation data để train models của họ (với API usage và trả phí). Điều này quan trọng với doanh nghiệp có dữ liệu nhạy cảm. Tuy nhiên, best practice: không paste sensitive data (PII, credentials, confidential contracts) trực tiếp vào prompt. Dùng references hoặc anonymize trước.

Cost management at scale

Khi team scale lên, chi phí API cũng scale. Chiến lược tối ưu cost:

Task Type Model Lý do
Simple Q&A, formatting, classification Haiku Nhanh, rẻ, đủ cho tasks đơn giản. ~20x rẻ hơn Opus.
Code generation, analysis, complex writing Sonnet Balance giữa capability và cost. Phù hợp 80% tasks.
Complex architecture, strategic analysis Opus Dùng khi thực sự cần. Không phải mọi task đều cần model mạnh nhất.

Prompt caching là tính năng tiết kiệm chi phí quan trọng nhất. Khi bạn gửi cùng một context dài (CLAUDE.md, system prompt, codebase context) nhiều lần, Anthropic cache nó — bạn chỉ trả cho phần thay đổi. Tiết kiệm đến 90% cost cho repeated context. Tự động với API, chỉ cần structure prompts đúng (stable context ở đầu, dynamic content ở cuối).

Batch API: Với workloads không cần real-time response (overnight processing, bulk analysis), dùng Batch API để có 50% discount. Phù hợp với: monthly report generation, bulk content analysis, large dataset processing.

Token budgets: Set spending limits per project, per team. Nhận alert khi gần limit. Với Enterprise có usage dashboards để track theo department.

Multi-project coordination

Monorepo với nhiều packages/apps có thể phức tạp với Claude Code. Strategies:

  • Module-level CLAUDE.md: Mỗi package có CLAUDE.md riêng với context của package đó. Claude tự động pick up context phù hợp dựa trên working directory.
  • Shared skills library: Cross-project skills ở root level, project-specific skills trong folder riêng. Claude biết dùng skill nào tùy context.
  • Dependency awareness: CLAUDE.md của từng package document interface của nó — "package này export gì, expect input gì." Claude Code hiểu cross-package dependencies khi refactor.

Disaster recovery

Claude Code làm việc với files — đây là rủi ro khi agent có thể modify nhiều files cùng lúc. Mitigation:

  • Git commit trước khi chạy: Mọi thay đổi của Claude dễ dàng revert bằng git checkout . hoặc git revert
  • Branch protection: Main branch chỉ accept merge qua PR đã review, không phải direct push — kể cả từ Claude
  • Permission restrictions: Claude không có permission để chạy destructive commands (drop database, delete production files) nếu không được explicitly allow
  • Audit trail: Git history là complete audit trail — ai thay đổi gì, lúc nào, tại sao
🔧
Thực hành

Tạo cost optimization plan cho team: (1) Categorize tasks team đang làm theo complexity — simple/medium/complex. (2) Assign model tier cho mỗi category. (3) Estimate monthly token usage và cost dựa trên current usage. (4) Identify top 3 opportunities cho prompt caching — những context nào được gửi lặp đi lặp lại? (5) Tính potential savings và present cho management với ROI calculation.
