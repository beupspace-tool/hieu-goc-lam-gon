Bài 2.4: Prompt hiệu quả

⏱ ~8 phút đọc Level 2
💡
Nguyên lý

Prompt tốt = kết quả tốt. Prompt không phải câu hỏi — nó là bản brief. Càng rõ ràng, Claude càng chính xác. Giống như brief cho nhân viên sáng tạo: bạn cho biết mục tiêu, khách hàng, giới hạn, và format mong muốn — thì bạn nhận lại đúng thứ cần.

Framework ICC: Cấu trúc prompt cơ bản

Mọi prompt tốt đều có 3 thành phần cốt lõi:

  • I — Instructions (Hướng dẫn): Bạn muốn CC làm gì? Cụ thể, rõ ràng. "Viết email" vs "Viết email chào hàng sản phẩm mới cho khách hàng cũ."
  • C — Context (Ngữ cảnh): Background cần thiết để CC hiểu tình huống. Ai là người đọc? Đây là đợt khuyến mãi nào? Đã có cuộc trao đổi nào trước chưa?
  • C — Constraints (Giới hạn): Yêu cầu bắt buộc và giới hạn. Độ dài, tone, format, những thứ không được làm.

Prompt kém: "Viết email marketing."

Prompt ICC tốt:

Viết email marketing chào hàng sản phẩm mới (áo dài truyền thống).
[Instructions]

Khách hàng là phụ nữ 30–50 tuổi đã mua hàng ít nhất 1 lần trong 6 tháng qua.
Họ quan tâm đến chất liệu và nguồn gốc sản phẩm.
[Context]

Giới hạn: tối đa 200 từ, tone thân mật như bạn bè, không dùng emoji,
kết thúc bằng CTA "Xem ngay" link về trang sản phẩm. Không đề cập giá.
[Constraints]

6 pattern prompt hiệu quả

Pattern 1
Explicit Role — Giao vai trò cụ thể

Nói với CC vai trò nó đang đảm nhận để nó điều chỉnh tone và cách tiếp cận phù hợp.

Viết bài giới thiệu sản phẩm.

"Bạn là content writer cho công ty phần mềm B2B. Viết bài giới thiệu sản phẩm quản lý nhân sự cho website — nhắm vào giám đốc nhân sự công ty 50–200 nhân viên."

Pattern 2
Step-by-step — Hướng dẫn từng bước

Khi task phức tạp, chia thành các bước rõ ràng. CC sẽ thực hiện tuần tự và có thể dừng hỏi nếu cần.

"Bước 1: Đọc file sales-q1.csv và hiểu cấu trúc dữ liệu. Bước 2: Tính tổng doanh thu theo từng tháng. Bước 3: Tìm top 3 sản phẩm bán chạy nhất. Bước 4: Tạo file report-q1-summary.md với kết quả."

Pattern 3
Example-driven — Cho ví dụ mẫu

Thay vì mô tả, cho CC xem bạn muốn kết quả trông như thế nào.

"Viết 3 caption Instagram cho sản phẩm mới. Theo phong cách này: 'Mỗi buổi sáng là một trang giấy trắng. Bắt đầu ngày của bạn với [sản phẩm] — nhẹ nhàng, tươi sáng, đúng nghĩa là của bạn.' Không dùng emoji, kết thúc bằng hashtag."

Pattern 4
Constraints first — Nêu giới hạn trước

Đặc biệt hữu ích khi bạn biết rõ những gì KHÔNG muốn hơn là những gì muốn.

"Tóm tắt cuộc họp. Yêu cầu: tối đa 150 từ, dạng bullet points, chỉ liệt kê action items và người chịu trách nhiệm, không cần ghi lại nội dung thảo luận chi tiết."

Pattern 5
Output format — Chỉ định format đầu ra

Nói rõ bạn muốn kết quả ở dạng gì để không phải format lại sau khi nhận.

"Phân tích file khach-hang.csv và trả về kết quả dưới dạng bảng Markdown với 3 cột: Tên, Số đơn hàng, Giá trị trung bình. Sắp xếp theo giá trị giảm dần."

Pattern 6
Iterative — Làm bản nháp trước, chỉnh sau

Với content quan trọng, đừng kỳ vọng bản đầu tiên là hoàn hảo. Dùng quy trình 2 bước.

"Tạo bản nháp email thông báo tăng giá dịch vụ. Đây là lần viết đầu — tôi sẽ review và cho phản hồi, sau đó bạn chỉnh sửa. Ưu tiên: thông điệp rõ ràng, tone không xin lỗi nhưng tôn trọng."

Ví dụ thực tế: Bad vs Good prompt

Tình huống 1: Phân tích báo cáo

Prompt kém: "Phân tích file này." — CC không biết file gì, phân tích để làm gì, kết quả trình bày thế nào.

Prompt tốt:

Đọc file bao-cao-ban-hang-thang-2.csv.
Tôi cần hiểu: (1) Tháng 2 doanh thu tổng bao nhiêu?
(2) Sản phẩm nào bán kém nhất? (3) So với tháng 1 trong file
bao-cao-ban-hang-thang-1.csv thì tăng hay giảm bao nhiêu %?
Trả kết quả dưới dạng 3 bullet points ngắn gọn.

Tình huống 2: Viết nội dung

Prompt kém: "Viết post Facebook quảng cáo."

Prompt tốt:

Bạn là social media manager cho tiệm bánh handmade nhỏ ở Hà Nội.
Viết post Facebook giới thiệu bánh macaron vị trà xanh mới ra lò.
Khách hàng: nữ 20–35 tuổi, thích đồ ăn đẹp, hay chụp ảnh.
Yêu cầu: 80–100 chữ, bắt đầu bằng câu hỏi gợi cảm xúc,
cuối gọi mọi người comment đặt hàng. 5 emoji phù hợp.

Battle-tested patterns từ power users

Pattern 7 — Tip #1 của cộng đồng
Plan Mode First — Phân tích trước, code sau

Đây là tip được nhắc đến nhiều nhất từ power users. Thay vì để CC bắt tay vào làm ngay, hãy bật Plan Mode trước: nhấn Shift+Tab 2 lần hoặc gõ /plan.

Trong Plan Mode, CC phân tích yêu cầu, đặt câu hỏi, và đề xuất approach — nhưng không sửa file nào. Bạn review plan, điều chỉnh nếu cần, rồi mới approve để CC thực thi.

Dùng trực tiếp: "Tạo landing page cho sản phẩm X." → CC bắt đầu code ngay, có thể đi hướng sai.

Dùng /plan trước: "Tạo landing page cho sản phẩm X." → CC hỏi về audience, stack, sections cần có → bạn confirm → CC execute đúng ngay lần đầu.

Nếu chỉ nhớ 1 điều từ guide này: Plan trước khi code.

Pattern 8
The Verification Pattern — Tự kiểm tra kết quả

Thay vì chỉ yêu cầu CC viết code, thêm vào cuối: "và viết tests cho nó, rồi chạy tests." CC tự verify output của mình — chất lượng tăng 2–3x so với chỉ viết code.

"Viết function tính thuế VAT."

"Viết function tính thuế VAT. Sau đó viết unit tests bao gồm edge cases (số âm, số 0, số rất lớn), và chạy tests để xác nhận tất cả pass."

Áp dụng cho bất kỳ task nào có thể verify: "tạo và kiểm tra", "viết và validate", "phân tích và xác nhận số liệu."

Pattern 9
Subagent Delegation — Giảm tải context chính

Khi cần research hoặc xử lý task phụ, thay vì nhồi hết vào một prompt, hãy nói rõ: "Research X và báo cáo lại." CC sẽ spawn subagent để xử lý riêng — giảm 40–60% context usage của conversation chính.

"Trước khi viết code, research xem Mailchimp API v3 có endpoint nào để bulk-unsubscribe không. Báo cáo lại rồi mới bắt đầu implement."

Ví dụ prompt đầy đủ từ thực tế

Bad → Good → Best: Tạo landing page

Bad: "Tạo landing page." — Không đủ context để làm gì.

Good:

Tạo landing page cho khóa học online "Thành thạo Excel trong 30 ngày",
audience là nhân viên văn phòng 22–35 tuổi muốn thăng tiến.
Cần: hero section với value proposition, 3 feature cards,
testimonial section, và CTA đăng ký email.
Dùng Tailwind CSS, responsive, dark mode. Chạy dev server sau khi xong.

Best: Gõ /plan trước → CC hỏi về màu sắc brand, số testimonials, CTA action cụ thể, integrations cần → bạn confirm → CC execute. Kết quả đúng ngay lần đầu, không cần revise nhiều.

Sai lầm thường gặp khi viết prompt

  • Quá mơ hồ: "Giúp tôi với email này" không có thông tin gì để CC làm việc.
  • Quá nhiều yêu cầu trong một prompt: Nếu cần 5 thứ khác nhau, gửi 5 prompt riêng. CC làm từng việc tốt hơn là làm 5 việc cùng lúc.
  • Mâu thuẫn giữa các yêu cầu: "Viết ngắn gọn nhưng đầy đủ mọi chi tiết" — ngắn gọn hay đầy đủ? Bạn phải chọn một.
  • Quên context: CC không nhớ cuộc hội thoại trước. Mỗi session mới, cần cung cấp lại ngữ cảnh quan trọng.
  • Không nói format đầu ra: Nếu bạn cần bảng Excel mà không nói, CC có thể trả về bảng text thuần. Nói rõ format bạn cần.
🔧
Thực hành

Viết 3 prompts theo framework ICC cho 3 tình huống thực tế: (1) Viết email chào hàng cho một sản phẩm/dịch vụ bạn đang bán, (2) Tóm tắt một cuộc họp gần đây theo dạng action items, (3) Phân tích một file dữ liệu bạn đang có. Sau khi viết xong, kiểm tra: mỗi prompt có đủ Instructions + Context + Constraints không?
