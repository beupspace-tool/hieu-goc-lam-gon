Bài 2.2: CLAUDE.md Templates

⏱ ~6 phút đọc Level 2
💡
Nguyên lý

Không cần bắt đầu từ zero. Template cho bạn khung chuẩn — bạn chỉ cần điền chi tiết vào chỗ trống. Giống như hợp đồng mẫu: luật sư đã soạn sẵn điều khoản tiêu chuẩn, bạn chỉ điền tên và số tiền.

Template cơ bản — Cho bất kỳ dự án nhỏ nào

Nếu bạn chưa biết bắt đầu từ đâu, hãy dùng template này. Phù hợp cho folder làm việc cá nhân, dự án một người, hoặc bất kỳ dự án nhỏ nào chưa có quy trình rõ ràng.

# CLAUDE.md

## Dự án
[Tên dự án] — [Mô tả 1–2 câu: dự án này làm gì, ai dùng nó]

## Ngôn ngữ & Tone
- Ngôn ngữ làm việc: [Tiếng Việt / Tiếng Anh / Song ngữ]
- Tone: [Thân thiện / Chuyên nghiệp / Kỹ thuật]
- Đơn vị: [VND / USD], định dạng ngày: [DD/MM/YYYY]

## Cấu trúc folder
[Mô tả ngắn gọn các folder chính]
/docs/         — Tài liệu
/data/         — Dữ liệu và file CSV
/templates/    — Template tái sử dụng
/reports/      — Báo cáo xuất ra

## Quy ước
- Tên file: [kebab-case / snake_case / PascalCase]
- [Quy tắc quan trọng khác của bạn]

## Ngữ cảnh
[Ai dùng? Vấn đề gì cần giải quyết? Ưu tiên gì?]

Template cho team — Khi làm việc nhóm

Khi có nhiều người cùng dùng Claude Code trong một dự án, CLAUDE.md cần nêu rõ quy ước chung để CC không làm khác nhau cho từng người.

# CLAUDE.md — [Tên dự án team]

## Dự án
[Mô tả dự án] — Team [X] người, [mục tiêu]

## Team & Phân quyền
- Lead: [Tên] — quyết định kiến trúc, code review
- Developer: [Tên] — implement features
- Content: [Tên] — nội dung, copywriting

## Tech Stack
- [Ngôn ngữ/Framework chính]
- [Database nếu có]
- [Tools CI/CD, hosting]

## Quy ước code
- Tên biến: camelCase
- Tên file: kebab-case
- Commit message: [feat/fix/docs/chore]: mô tả ngắn
- Branch: feature/[tên], fix/[tên]

## Quy trình làm việc
1. Tạo branch từ main
2. Implement và test
3. Tạo Pull Request → lead review
4. Merge sau khi được approve

## Lệnh thường dùng
- Chạy local: [lệnh]
- Chạy test: [lệnh]
- Build: [lệnh]
- Deploy: [lệnh]

## Không được làm
- Commit trực tiếp vào main
- Push code chưa qua test
- Xóa file mà không backup

Template cho backoffice — Quản lý nội bộ

Dành cho các dự án xử lý dữ liệu nội bộ: quản lý đơn hàng, kho, khách hàng, tài chính. Không cần kỹ năng kỹ thuật — chỉ cần mô tả rõ bạn đang quản lý gì.

# CLAUDE.md — Backoffice [Tên công ty]

## Tổng quan
Quản lý dữ liệu nội bộ cho [Tên công ty].
Bao gồm: [đơn hàng / khách hàng / kho / tài chính / nhân sự].

## Dữ liệu đang quản lý
- customers.csv: Danh sách khách hàng (email, tên, điện thoại, địa chỉ)
- orders.csv: Đơn hàng (mã đơn, ngày, khách hàng, sản phẩm, giá trị)
- products.csv: Catalog sản phẩm (mã, tên, giá, tồn kho, danh mục)
- [File khác nếu có]

## Cấu trúc folder
/data/          — File CSV chính
/reports/       — Báo cáo đã tạo
/backups/       — Backup trước khi sửa
/templates/     — Template báo cáo

## Quy ước dữ liệu
- Ngày: DD/MM/YYYY
- Tiền: VND, không dấu chấm phân cách nghìn trong CSV
- Tên: Họ tên đầy đủ, không viết tắt
- Email: chữ thường

## Báo cáo cần tạo thường xuyên
- Báo cáo tuần: doanh thu, đơn hàng mới, tồn kho thấp
- Báo cáo tháng: tổng doanh thu, top sản phẩm, khách hàng mới

## Lưu ý an toàn dữ liệu
- Luôn backup trước khi sửa file CSV lớn
- Không xóa dữ liệu cũ — archive vào /backups thay thế
- [Quy tắc bảo mật cụ thể của công ty]

Template cho dự án Claude Code — Meta-template

Dành cho ai dùng Claude Code để phát triển phần mềm, website, hoặc bất kỳ dự án có code. Đây là template tối giản nhất — mọi dòng đều thiết yếu, không có gì thừa. Được tạo tự động khi bạn chạy /init, nhưng biết cấu trúc này giúp bạn tùy chỉnh tốt hơn.

# CLAUDE.md

## Project
- Name: [project name]
- Stack: [tech stack — framework, database, language, versions]
- URL: [nếu đã deploy]

## Commands
- `npm run dev` — start dev server
- `npm test` — run tests
- `npm run build` — production build
- `npm run lint` — check code style

## Structure
[brief folder map — 5–8 dòng, chỉ folder quan trọng]
src/
  components/   — UI components
  pages/        — page routes
  utils/        — helper functions
public/         — static assets

## Conventions
- Tên file: kebab-case
- Tên component: PascalCase
- Commit: feat/fix/docs/chore: mô tả ngắn
- [Quy tắc quan trọng khác]

## Context
- [Ai dùng sản phẩm này?]
- [Vấn đề gì đang giải quyết?]
- [Phase hiện tại / priority?]
💡
Nguyên tắc sizing

Anthropic khuyến nghị CLAUDE.md dưới 300 dòng tổng. Cộng đồng power users nhận thấy dưới 150 dòng hướng dẫn cho kết quả tốt nhất — Claude tập trung hơn, ít bị "scatter" hơn. Chạy /init để tạo file khởi đầu, rồi dọn dẹp xuống dưới 150 dòng. Mỗi dòng phải kiếm được chỗ của nó.

Cách sử dụng template

  1. Chọn template phù hợp nhất với loại công việc của bạn (hoặc kết hợp 2 template nếu cần).
  2. Copy nội dung template vào file mới tên CLAUDE.md.
  3. Điền thông tin thực tế vào tất cả chỗ có dấu [...]. Xóa phần nào không liên quan.
  4. Đặt file vào thư mục gốc dự án — cùng cấp với folder con, không phải trong subfolder.
  5. Mở Claude Code và thực hiện yêu cầu đầu tiên. Quan sát xem CC có đọc và áp dụng context không.
⚠️
Cảnh báo

Template là điểm khởi đầu, không phải đích đến. Một CLAUDE.md tốt là file được cập nhật thường xuyên — khi dự án thay đổi, khi bạn phát hiện CC hiểu sai điều gì đó, hoặc khi bạn muốn thêm quy tắc mới. Đừng bao giờ copy template rồi không chỉnh sửa gì cả.

🔧
Thực hành

Chọn 1 template phù hợp nhất với công việc của bạn. Tùy chỉnh bằng cách điền thông tin thực tế vào tất cả phần [...]. Đặt vào folder dự án và mở Claude Code. Thực hiện 2–3 yêu cầu khác nhau và quan sát xem CC có áp dụng đúng ngữ cảnh từ CLAUDE.md không.
