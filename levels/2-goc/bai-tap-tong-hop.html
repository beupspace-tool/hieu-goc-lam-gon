Bài 2.9: Bài tập tổng hợp

⏱ ~7 phút đọc Level 2 Thực hành
💡
Mục đích bài này

6 tình huống thực tế kết hợp tất cả kỹ năng Level 2. Chọn ít nhất 2 tình huống gần nhất với công việc của bạn và thực hiện thật sự — không chỉ đọc qua. Kỹ năng chỉ hình thành khi bạn làm, không phải khi đọc.

S1
Khởi tạo dự án marketing
2.1 CLAUDE.md 2.2 Templates 2.3 Cấu trúc
Mục tiêu

Tạo folder dự án marketing hoàn chỉnh cho 1 sản phẩm/dịch vụ bạn đang kinh doanh — sẵn sàng để CC làm việc hiệu quả ngay từ đầu.

Bước thực hiện
  1. Tạo folder marketing-[ten-san-pham]/
  2. Tạo cấu trúc con: /emails/, /social/, /blog/, /briefs/, /templates/
  3. Tạo CLAUDE.md từ template marketing (bài 2.2) — điền thông tin thực của doanh nghiệp bạn
  4. Tạo brief-chien-dich-thang-4.md với mục tiêu content tháng 4
  5. Yêu cầu CC đề xuất lịch nội dung 4 tuần dựa trên brief
Kết quả mong đợi

Folder marketing được tổ chức rõ ràng, CLAUDE.md phản ánh đúng thương hiệu của bạn, và có lịch content 4 tuần cụ thể để bắt đầu thực hiện.

Gợi ý

Đừng hoàn hảo hóa CLAUDE.md ngay từ đầu. Viết phiên bản đầu tiên trong 15 phút, rồi cập nhật dần khi bạn phát hiện CC hiểu sai điều gì đó.

S2
Dọn dẹp danh sách khách hàng
2.5 File 2.7 Dữ liệu 2.4 Prompt
Mục tiêu

Biến danh sách khách hàng lộn xộn thành dataset sạch, có phân loại, có báo cáo tóm tắt — trong dưới 30 phút.

Bước thực hiện
  1. Xuất danh sách khách hàng từ phần mềm hoặc tạo file customers-raw.csv mẫu (ít nhất 20 dòng)
  2. Yêu cầu CC đọc file và báo cáo vấn đề: trùng lặp, thiếu thông tin, format không nhất quán
  3. Yêu cầu CC dọn dẹp: xóa trùng, chuẩn hóa tên, fix format — lưu vào customers-clean.csv
  4. Yêu cầu CC phân loại theo tiêu chí bạn chọn (tỉnh thành, giá trị mua, tần suất mua)
  5. Yêu cầu CC tạo báo cáo tóm tắt customer-insights.md
Kết quả mong đợi

File khách hàng sạch, phân loại rõ ràng, và báo cáo tóm tắt có thể dùng để quyết định chiến lược marketing.

Gợi ý

Nếu chưa có data thật, dùng prompt này: "Tạo file customers-raw.csv mẫu với 25 dòng, cố ý thêm một số vấn đề dữ liệu để tôi luyện tập dọn dẹp: 3 email trùng, 5 tên không nhất quán về case, 2 dòng thiếu tỉnh thành."

S3
Content week — Một tuần nội dung hoàn chỉnh
2.6 Nội dung 2.4 Prompt 2.5 File
Mục tiêu

Tạo toàn bộ nội dung cho 1 tuần — 3 bài blog hoặc 7 caption social — theo đúng quy trình 5 bước.

Bước thực hiện
  1. Tạo brief-tuan-[X].md với theme tuần, audience, tone, số lượng bài
  2. Yêu cầu CC đề xuất 5 ý tưởng nội dung cho tuần — bạn chọn 3
  3. Với mỗi ý tưởng được chọn: CC viết bản nháp → bạn review → CC revise
  4. Lưu bản cuối mỗi bài vào /content/tuan-[X]/
  5. Yêu cầu CC tạo file schedule-tuan-[X].md với lịch đăng cụ thể
Kết quả mong đợi

Toàn bộ nội dung 1 tuần được chuẩn bị trước, tổ chức trong folder, sẵn sàng đăng theo lịch.

Gợi ý

Bắt đầu với social media (ngắn, nhanh feedback) trước khi thử blog. Mỗi vòng review chỉ cần 3–5 điểm cụ thể — đừng viết nhận xét dài hơn bài gốc.

S4
Báo cáo cuối tháng
2.7 Dữ liệu 2.8 Báo cáo 2.5 File
Mục tiêu

Tạo báo cáo tháng executive-level từ nhiều nguồn dữ liệu khác nhau — tự động thay vì thủ công.

Bước thực hiện
  1. Thu thập data: doanh thu (CSV), đơn hàng (CSV), note tay về sự kiện quan trọng (TXT)
  2. Yêu cầu CC đọc tất cả file và tổng hợp số liệu chính
  3. Yêu cầu CC so sánh với tháng trước (nếu có file tháng trước)
  4. Yêu cầu CC viết báo cáo executive summary — tối đa 1 trang A4
  5. Review và bổ sung nhận xét cá nhân của bạn (insight mà data không thể hiện)
  6. Lưu vào /reports/monthly/report-[thang]-[nam].md
Kết quả mong đợi

Báo cáo chuyên nghiệp, có số liệu cụ thể, có so sánh xu hướng, và có nhận xét từ bạn — sẵn sàng gửi cho sếp hoặc đối tác.

Gợi ý

Tạo file report-template.md một lần với cấu trúc chuẩn — CC sẽ điền vào template đó thay vì tạo format mới mỗi tháng. Consistency quan trọng hơn là mỗi tháng một format đẹp hơn.

S5
Email campaign preparation
2.8 Email 2.2 Templates 2.7 Dữ liệu
Mục tiêu

Chuẩn bị đầy đủ cho một email campaign — templates, phân khúc khách hàng, lịch gửi — trong một buổi làm việc.

Bước thực hiện
  1. Tạo campaign-brief.md: mục tiêu campaign, offer, timeline, KPIs
  2. Yêu cầu CC phân tích customers.csv và gợi ý 3 phân khúc để target khác nhau
  3. Yêu cầu CC tạo 3 email templates khác nhau cho 3 phân khúc — một offer nhưng angle khác nhau
  4. Review và chỉnh sửa từng email theo feedback cụ thể
  5. Yêu cầu CC tạo campaign-schedule.md: email nào gửi ngày nào, cho segment nào, subject line A/B test
  6. Tổ chức tất cả vào folder /campaigns/[ten-campaign]/
Kết quả mong đợi

Campaign folder hoàn chỉnh: brief, 3 email templates theo phân khúc, lịch gửi — sẵn sàng import vào tool email marketing.

Gợi ý

Sau khi gửi campaign thật, ghi lại kết quả (open rate, click rate) vào file. Lần campaign sau, cho CC đọc kết quả cũ để học và đề xuất cải thiện. Data từ thực tế là input quý giá nhất.

S6
Setup hoàn chỉnh Claude Code cho dự án mới
2.1 CLAUDE.md 2.3 Cấu trúc 2.4 Prompt /plan /cost /compact
Mục tiêu

Thực hành toàn bộ workflow khởi tạo dự án mới với Claude Code — từ zero đến setup hoàn chỉnh, áp dụng tất cả kỹ năng Level 2 trong một luồng liên tục.

Bước thực hiện
  1. Tạo folder my-project (qua Finder/File Explorer hoặc terminal). Mở trong Claude Code Desktop App hoặc VS Code.
  2. Nói với Claude: "Khởi tạo git repository cho project này" — Claude sẽ tự chạy lệnh cần thiết.
  3. Chạy /init — CC tự động tạo CLAUDE.md từ cấu trúc hiện tại
  4. Review CLAUDE.md: xóa những gì không cần, thêm business context, kiểm tra dưới 150 dòng
  5. Gõ /plan rồi mô tả project — CC lên kế hoạch cấu trúc dự án, bạn review và approve
  6. Execute plan — CC tạo cấu trúc folder, file khởi đầu
  7. Gõ /cost — kiểm tra đã dùng bao nhiêu token và chi phí ước tính
  8. Gõ /compact — nén context để tiếp tục làm việc hiệu quả hơn
  9. Nói với Claude: "Commit tất cả thay đổi với message 'feat: initial project setup'" — Claude tự chạy git commit.
Kết quả mong đợi

Dự án mới có CLAUDE.md tối ưu, cấu trúc folder rõ ràng, git khởi tạo, và bạn đã trải qua đầy đủ vòng lặp: init → plan → execute → check cost → compact → commit.

Tại sao scenario này quan trọng

Đây là workflow bạn sẽ lặp lại mỗi khi bắt đầu dự án mới. Làm quen với thứ tự /init → /plan → execute → /cost → /compact → commit giúp bạn làm việc có hệ thống thay vì mò mẫm. Scenario này kết hợp: CLAUDE.md (2.1), templates (2.2), project structure (2.3), và prompt hiệu quả (2.4) trong một luồng thực tế.

Level 2 hoàn thành!

Bạn đã có kỹ năng cốt lõi để vận hành hàng ngày với Claude Code. CLAUDE.md, prompt hiệu quả, làm việc với file, quản lý content, data, email và báo cáo — đây là những công cụ bạn sẽ dùng mỗi ngày.

Level 3 sẽ nâng cấp từ task lẻ lên QUY TRÌNH — MCP (kết nối CC với dịch vụ bên ngoài), Git (version control), Skills (mở rộng khả năng CC), và tự động hóa theo team. Bạn đã có nền tảng vững để bước tiếp.
