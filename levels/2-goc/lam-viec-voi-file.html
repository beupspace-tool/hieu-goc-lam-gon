Bài 2.5: Làm việc với file

⏱ ~7 phút đọc Level 2
💡
Nguyên lý

Claude Code mạnh nhất khi làm việc với file thật trên máy tính của bạn. Đọc, tạo, sửa, tìm kiếm — đây là sức mạnh mà Claude Chat (phiên bản web) hoàn toàn không có. Khi bạn đưa CC vào folder dự án, bạn đang cho nó "tay chân" để hành động, không chỉ "miệng" để nói.

Các loại thao tác với file

📖
Đọc & Phân tích file

CC có thể đọc và hiểu nội dung của hầu hết các loại file văn bản: .txt, .md, .csv, .json, .html, .py, .js, v.v.

Ví dụ prompt:

Đọc file report-thang-2.csv và tóm tắt:
- Có bao nhiêu dòng dữ liệu?
- Các cột là gì?
- Số liệu nào có giá trị lớn nhất và nhỏ nhất?
Đọc file meeting-notes.md và liệt kê
tất cả action items cùng với người chịu trách nhiệm.
✏️
Tạo file mới

CC có thể tạo file mới với nội dung bạn yêu cầu — trong bất kỳ format nào.

Tạo file meeting-template.md trong folder /templates/
với template họp chuẩn bao gồm:
- Ngày và giờ họp
- Danh sách người tham dự
- Nội dung thảo luận (dạng bullet)
- Quyết định đã đưa ra
- Action items: người làm, deadline
Tạo file weekly-report-template.md với cấu trúc
báo cáo tuần: tổng quan, thành tựu, vướng mắc,
kế hoạch tuần tới, metrics chính.
🔧
Sửa file hiện có

CC có thể mở và chỉnh sửa nội dung file đang có — thêm, xóa, sửa đổi nội dung theo yêu cầu của bạn.

Mở file prices.csv, tăng tất cả giá trị
trong cột "Giá" lên 10%. Lưu lại file.
Sửa file email-chao-hang.md:
thay tên khách hàng "[Tên]" thành "Chị Mai",
thay tên sản phẩm "[Sản phẩm]" thành "Gói Premium".
🔍
Tìm kiếm xuyên file

CC có thể tìm kiếm từ khóa, pattern, hoặc nội dung cụ thể across nhiều file trong folder.

Tìm tất cả file trong folder /reports/
có chứa từ "overdue" hoặc "quá hạn".
Liệt kê tên file và dòng chứa từ đó.
Tìm trong /data/customers/ xem có
email nào bị trùng nhau không.
Liệt kê các email trùng nếu có.
⚡
Batch operations — Xử lý hàng loạt

Đây là nơi CC thực sự tiết kiệm hàng giờ công sức. Những thứ bạn phải làm thủ công lặp đi lặp lại, CC có thể làm trong vài giây.

Trong folder /photos/, đổi tên tất cả file
theo format YYYY-MM-DD-[tên gốc].
Ví dụ: "IMG_1234.jpg" → "2026-03-15-IMG_1234.jpg"
Đọc tất cả file .md trong folder /blog/
và tạo file index.md liệt kê:
- Tên file, tiêu đề bài (dòng đầu tiên), ngày tạo
Chuyển đổi file customers.csv thành
bảng Markdown và lưu vào customers-table.md

An toàn khi làm việc với file

Claude Code hiểu rằng thao tác file là không thể undo dễ dàng. CC có một số cơ chế bảo vệ tự động:

  • CC xin phép trước khi xóa. Nếu bạn yêu cầu xóa file, CC sẽ hỏi xác nhận, không tự xóa ngay.
  • CC mô tả kế hoạch trước khi thực hiện. Với thao tác phức tạp hoặc ảnh hưởng nhiều file, CC thường mô tả sẽ làm gì trước khi thực sự làm.
  • CC đề xuất backup trước thao tác lớn. Khi sửa file quan trọng, CC thường gợi ý tạo bản backup.
⚠️
Cảnh báo quan trọng

Trước khi yêu cầu CC thực hiện thao tác hàng loạt trên file quan trọng (đổi tên, sửa, xóa nhiều file cùng lúc): hãy backup folder đó trước. Copy sang folder khác hoặc Google Drive. CC làm rất nhanh và chính xác — nhưng nếu bạn ra lệnh sai, không thể undo trong một nốt nhạc.

Loại file nào CC làm được?

  • Có thể đọc và sửa: .txt, .md, .csv, .json, .html, .css, .js, .py, .xml, .yaml
  • Có thể đọc nội dung text: Nhiều loại file khác nếu chứa text thuần
  • Không làm được trực tiếp: .xlsx (Excel), .docx (Word), .pdf — CC cần bạn xuất sang CSV/TXT trước, hoặc dùng công cụ chuyển đổi
🔧
Thực hành

Tạo folder test-files/ và tạo 5 file mẫu bên trong (có thể là file text với nội dung bất kỳ). Sau đó yêu cầu CC: (1) Liệt kê tất cả file trong folder và mô tả ngắn nội dung mỗi file, (2) Đổi tên tất cả file theo quy tắc kebab-case, (3) Tạo file tóm tắt summary.md liệt kê tên và nội dung ngắn của tất cả file.
