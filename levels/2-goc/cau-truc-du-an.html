Bài 2.3: Cấu trúc dự án

⏱ ~6 phút đọc Level 2
💡
Nguyên lý

Cấu trúc folder tốt giúp Claude Code hiểu dự án nhanh hơn và đưa ra kết quả chính xác hơn. Giống như văn phòng ngăn nắp — khách đến biết ngay tìm gì ở đâu mà không cần hỏi. Tổ chức tốt = giao tiếp tốt.

Tại sao cấu trúc folder quan trọng với Claude Code?

Claude Code không "nhìn" dự án của bạn như con người. CC đọc tên file và folder để hiểu ngữ cảnh. Khi bạn yêu cầu CC làm gì đó, việc đầu tiên CC làm là khám phá cấu trúc folder để định hướng.

Ví dụ, bạn nói: "Tạo email thông báo khuyến mãi cho tháng 3."

  • Folder tốt: CC thấy /marketing/emails/ — biết ngay nên đặt file mới ở đây, và tham khảo các email cũ trong folder để giữ đúng format.
  • Folder lộn xộn: CC thấy /random-stuff/ và /file-cu/ — không biết đặt ở đâu, không có template tham khảo, kết quả chung chung.

Tên file và folder tốt = CC hiểu không cần giải thích. Đây là dạng "giao tiếp ngầm" hiệu quả nhất.

Quy tắc đặt tên

Hai nguyên tắc cần nhớ:

1. Dùng kebab-case (dấu gạch ngang)

Tên file và folder nên dùng chữ thường và dấu gạch ngang thay vì khoảng trắng hoặc chữ hoa. Lý do: khoảng trắng trong tên file gây lỗi terminal; camelCase khó đọc với người không quen code.

Không nên
Báo cáo Q1 2026.xlsx
EmailMarketing_FINAL_v3.docx
KhachHang_Nam2025.csv
file1.txt
Nên dùng
bao-cao-q1-2026.xlsx
email-marketing-thang-3.docx
khach-hang-2025.csv
danh-sach-san-pham.txt

2. Tên mô tả được nội dung

Đọc tên file, bạn — và Claude Code — phải biết ngay file này chứa gì mà không cần mở ra. Đây là nguyên tắc "self-documenting".

  • file1.csv → Không có thông tin gì
  • data.csv → Vẫn không biết data gì
  • khach-hang-ho-chi-minh-2025.csv → Biết ngay: danh sách khách hàng HCM năm 2025

Cấu trúc folder mẫu cho doanh nghiệp nhỏ

Đây là cấu trúc phổ biến và hiệu quả cho doanh nghiệp không có đội kỹ thuật:

my-business/
├── CLAUDE.md                    ← Sổ tay vận hành cho CC
├── README.md                    ← Ghi chú nhanh cho bản thân
│
├── marketing/
│   ├── emails/                  ← Template và nội dung email
│   ├── social/                  ← Nội dung mạng xã hội
│   │   ├── facebook/
│   │   └── instagram/
│   └── blog/                    ← Bài viết blog
│
├── reports/                     ← Báo cáo định kỳ
│   ├── weekly/
│   └── monthly/
│
├── data/                        ← Dữ liệu thô
│   ├── customers/               ← Danh sách khách hàng
│   ├── products/                ← Catalog sản phẩm
│   └── orders/                  ← Lịch sử đơn hàng
│
└── templates/                   ← Template tái sử dụng
    ├── email-chao-hang.md
    ├── bao-cao-tuan.md
    └── hop-dong-dich-vu.md

Cấu trúc folder cho website

Nếu dự án của bạn là website, đây là cấu trúc chuẩn:

my-website/
├── CLAUDE.md
├── index.html                   ← Trang chủ
│
├── src/                         ← Source code (do CC chỉnh sửa)
│   ├── pages/                   ← Các trang HTML
│   ├── components/              ← Thành phần tái sử dụng
│   └── scripts/                 ← JavaScript
│
├── assets/                      ← Tài nguyên tĩnh
│   ├── css/                     ← Style
│   ├── images/                  ← Hình ảnh
│   └── fonts/                   ← Font chữ
│
├── content/                     ← Nội dung (blog, text)
└── docs/                        ← Tài liệu dự án

Mẹo tổ chức thực tế

  • Một folder = một mục đích. Đừng nhồi nhiều loại file khác nhau vào cùng một folder. Khi CC tìm kiếm, nó nhìn vào tên folder để quyết định "vào hay không vào".
  • Tạo README.md trong mỗi folder chính. Chỉ cần 2–3 dòng mô tả folder này chứa gì và quy tắc gì. Ví dụ: "Folder này chứa tất cả email marketing. Tên file: email-[loại]-[tháng-năm].md"
  • Giữ đường dẫn ngắn. Tránh tạo quá nhiều tầng folder lồng nhau. Quá 3–4 tầng bắt đầu khó quản lý. Thay vì /data/old/backup/2024/q1/customers/, hãy dùng /archive/customers-2024/.
  • Archive thay vì xóa. Tạo folder /archive/ để dời file cũ vào thay vì xóa. CC không thể recover file đã xóa.

Best practices cho project root (từ cộng đồng Claude Code)

Ngoài cấu trúc folder, những file và thư mục ở root có vai trò đặc biệt với Claude Code:

  • CLAUDE.md ở root — Luôn đặt ở root, không trong subfolder. CC tìm ở đây trước tiên.
  • .claude/ directory — Thư mục cho project-specific settings của Claude Code. Đây là nơi CC lưu cài đặt và permissions.
  • .claude/settings.local.json — Permissions cho CC: file nào được phép đọc/sửa, lệnh nào được phép chạy. Không commit file này lên git.
  • plans/ directory — Nơi lưu implementation plans khi dùng CC cho dự án phức tạp. Cộng đồng khuyến nghị tạo thư mục này để CC có chỗ "suy nghĩ" có tổ chức.
  • Tên file dài và mô tả — Với LLM tools (Grep, Glob, Search), tên file dài như email-marketing-chao-hang-khach-hang-cu.md tốt hơn email.md. File tự mô tả chính nó.

Git Worktrees: chạy nhiều Claude Code session song song

Power users chạy 5–20 Claude Code sessions cùng lúc. Bí quyết: git worktrees — mỗi session trong một worktree riêng biệt, không có merge conflict.

Tại sao cần worktrees? Nếu 2 sessions cùng làm việc trên một folder, chúng có thể sửa cùng file gây conflict. Worktree tạo ra bản copy riêng của codebase cho mỗi branch, mỗi session làm việc độc lập.

Ví dụ thực tế: Thay vì làm tuần tự (feature A xong mới làm B), bạn chạy song song:

# Tạo worktree cho feature authentication
git worktree add ../my-project-auth -b feature/auth

# Tạo worktree cho feature dashboard
git worktree add ../my-project-dashboard -b feature/dashboard

# Mở 2 Claude Code sessions riêng biệt:
# Terminal 1: cd ../my-project-auth && claude
# Terminal 2: cd ../my-project-dashboard && claude

Kết quả: task mất 30 phút tuần tự → 5–10 phút khi chạy song song. Mỗi session có CLAUDE.md, context, và file riêng — không ảnh hưởng lẫn nhau.

Khi xong, merge về branch chính như bình thường: git merge feature/auth.

⚠️
Cảnh báo

Đừng cố tạo cấu trúc "hoàn hảo" ngay từ đầu. Bắt đầu đơn giản với 3–4 folder, rồi thêm dần khi cần. Cấu trúc quá phức tạp mà không dùng đến còn tệ hơn cấu trúc đơn giản nhưng thực tế.

🔧
Thực hành

Nhìn vào folder làm việc hiện tại của bạn. Liệt kê 3 thứ cần cải thiện: tên file nào khó hiểu, folder nào lộn xộn, file nào đặt sai chỗ. Sau đó yêu cầu Claude Code: "Đây là danh sách file trong folder [tên]. Đề xuất cách tổ chức lại theo quy tắc kebab-case và phân loại hợp lý." Xem CC gợi ý gì.
